# GLOBE-USV：带可证界与事件驱动重规划的异构多无人艇全局联合覆盖算法

> 文档性质：算法设计稿，不包含程序实现。
>
> 英文全称：Global Lexicographic Optimization with Bounds and Event-driven Replanning for Heterogeneous USVs。
>
> 目标任务：在障碍、海流、异构航速/传感器幅宽/续航约束和艇间冲突同时存在时，联合决定“由谁覆盖、按什么顺序覆盖、采用哪一种覆盖模式、何时通过共享水域”，并给出可行解、下界和可解释的最优性差距。
>

## 摘要

现有论文的主要问题并不是单艇旅行商问题求解得不够精细，而是把多艇任务拆成了连续但相互割裂的步骤：先分区或分配，再对每艘艇独立求旅行商路径，最后用局部避碰修补。前一步一旦固定，后一步无法跨艇交换任务；单艇路径即使分别达到最优，也不意味着舰队最大完工时间、总能耗或冲突等待达到较优，更不能据此给出全局最优性差距。

本文提出 GLOBE-USV。算法首先把待覆盖区域表示为可验证的原子覆盖需求，并为每个原子区域生成多个与无人艇型号、海流场和进出端点相关的候选覆盖模式。随后将一条艇级完整航线作为一个“路线列”，在同一个集合覆盖主问题中联合选择任务归属、访问顺序、覆盖方向、进出模式和等待策略。小中规模实例由分支—定价—割求解，可在有限候选模型上给出全局最优解或严格最优性差距；大规模实例由对偶引导的自适应大邻域搜索与路线池重组求解，并通过根节点松弛、工作量松弛和返航下界报告可验证的近优程度。连续时空冲突由冲突检测—切割—安全时隙列生成闭环处理。环境变化或无人艇故障发生时，算法冻结已执行前缀，仅对剩余覆盖需求进行事件驱动滚动重规划。

算法的核心贡献不是再设计一个单艇 TSP 元启发式，而是建立一个可同时容纳异构性、覆盖模式、舰队级目标、连续冲突和在线恢复的统一优化接口，并用“精确引擎 + 可扩展引擎 + 可证下界”使算法质量可以被证实，而不只依赖与若干启发式算法的均值比较。

## 1. 研究缺口与设计结论

### 1.1 为什么“每艘艇分别优化”不能推出“舰队整体较优”

设两艘艇的路线分别为 \(R_1,R_2\)，单艇子问题是在已经给定任务集合 \(A_1,A_2\) 后最小化各自路线：

\[
R_k^*=\arg\min_{R_k:\,\operatorname{cover}(R_k)=A_k} C_k(R_k).
\]

它只说明在固定分配 \(A_1,A_2\) 下，\(R_k^*\) 可能是较优的。真正的多艇问题则是：

\[
\min_{A_1,\ldots,A_K,R_1,\ldots,R_K}
\left[
\max_k T_k,\;
\sum_k E_k,\;
\text{turns},\;
\text{overlap},\;
\text{conflicts}
\right],
\]

其中任务集合本身也是决策变量，并且 \(R_k\) 的代价会受任务归属、覆盖方向、海流、其他艇的时刻表和共享航道影响。因此：

1. 固定分区会丢失跨艇任务交换所带来的改进；
2. 最短总距离不等于最小最大完工时间；
3. 欧氏距离下的好分配可能在障碍、转弯半径和海流下很差；
4. 独立路径在加入等待和避碰后会改变原有成本排序；
5. “最优单艇 TSP 的并集”一般不是多旅行商、车辆路径或多智能体覆盖问题的最优解。

所以，当前实验即使表明单艇 TSP 模块优于其他 TSP 算法，也只能证明局部模块有效；如果没有联合基线、下界、精确小实例和最优性差距，就不能证明整个多艇方案是较优解。

### 1.3 近三年高水平外部工作给出的直接启发

以下外部论文均位于 2023—2025 年的高水平机器人、人工智能期刊或主流顶级会议范围内；这里只借鉴其可核验的算法思想，不声称它们已经解决本文的海流异构多 USV 联合问题。

1. Tang 与 Ma，2023，IEEE Robotics and Automation Letters，[*Mixed Integer Programming for Time-Optimal Multi-Robot Coverage Path Planning with Efficient Heuristics*](https://arxiv.org/abs/2306.17609)：将 MCPP 归约为最小—最大根树覆盖并建立 MIP，能够报告分支定界下界；本文借鉴“用精确模型和下界评价近优性”，但将树覆盖扩展为多模式、异构、能量与时空冲突路线模型。
2. Tang 与 Ma，2024，AAAI，[*Large-Scale Multi-Robot Coverage Path Planning via Local Search*](https://ojs.aaai.org/index.php/AAAI/article/view/29707)：直接在实际覆盖路径上做局部搜索，并以 makespan 为核心指标；本文借鉴“优化真实路径而非只优化上游分区代理量”，并扩展为跨艇任务、模式和时刻表的大邻域。
3. Tang 与 Ma，2024，ICAPS，[*Multi-Robot Connected Fermat Spiral Coverage*](https://ojs.aaai.org/index.php/ICAPS/article/view/31520)：以等距线图和 MMRTC 处理不规则障碍区，兼顾 makespan、曲率和重叠；本文将 MCFS 作为候选覆盖骨架和暖启动之一，而不是最终固定分配。
4. Tang、Mao 与 Ma，2025，IEEE Transactions on Robotics，[*Large-Scale Multirobot Coverage Path Planning on Grids With Path Deconfliction*](https://www.ieee-ras.org/news/large-scale-multirobot-coverage-path-planning-on-grids-with-path-deconfliction/)：用 ESTC、路径级局部搜索和 MAPF 后处理同时关注大规模覆盖与路径消冲突；本文借鉴“覆盖规划必须经过显式多智能体冲突处理”，并把冲突代价进一步反馈到主优化问题。
5. Foster 等，2025，ICRA，[*An Efficient NSGA-II-Based Algorithm for Multi-Robot Coverage Path Planning*](https://doi.org/10.1109/ICRA55743.2025.11128792)：通过 donation mutation、multiple-parent crossover 和精英路径库搜索多目标 Pareto 解；本文借鉴跨机器人捐赠/交换和路径池概念，但以词典序目标和下界替代只报告 Pareto 优势。
6. Lu 等，2023，IEEE Robotics and Automation Letters，[*TMSTC*: A Path Planning Algorithm for Minimizing Turns in Multi-Robot Coverage*](https://doi.org/10.1109/LRA.2023.3293319)：在 STC 框架中直接减少多机器人转弯；本文把转弯作为真实航时/能耗的一部分并保留显式三级指标。
7. 2025 年 Artificial Intelligence，[*A scalable multi-robot goal assignment algorithm for minimizing mission time followed by total movement cost*](https://www.sciencedirect.com/science/article/pii/S0004370225001079)：先最小化 makespan，再在不破坏最优 makespan 的条件下最小化总移动成本；本文借鉴这种词典序原则，避免任意权重掩盖舰队瓶颈。

由上述工作可得三个设计原则：

1. 评价对象必须是最终可执行多艇路径，而不是分区面积、种子点距离或单艇 TSP 适应度；
2. 主目标应优先消除舰队瓶颈，同时保留总能耗、转弯、覆盖重叠和冲突等待；
3. 顶级论文不仅要给出更低的均值，还要给出精确小实例、下界、最优性差距、消融和物理实验。

## 2. 问题定义

### 2.1 环境、舰队与不确定性

待覆盖自由水域为

\[
\Omega_f=\Omega\setminus\left(\mathcal O\oplus B(r_{\text{safe}})\right),
\]

其中 \(\Omega\) 是任务区域，\(\mathcal O\) 是岛礁、浅滩和禁航区，\(\oplus B(r_{\text{safe}})\) 表示按照艇体尺寸、定位误差和控制误差进行安全膨胀。

舰队为 \(\mathcal K=\{1,\ldots,K\}\)。无人艇 \(k\) 具有：

\[
\Theta_k=
\{p_k^0,\psi_k^0,v_k^{\min},v_k^{\max},
\omega_k^{\max},\rho_k,w_k,B_k,P_k(\cdot),\mathcal C_k\},
\]

分别表示起点与艏向、速度范围、最大转艏率、最小转弯半径、传感器有效幅宽、可用电量、功率模型及可执行任务类型。

用有限情景集 \(\mathcal S\) 表示海流和环境预测不确定性。情景 \(s\) 的流场为 \(\mathbf c_s(p,t)\)，概率为 \(p_s\)。艇的简化运动学为

\[
\dot p_k=
v_k[\cos\psi_k,\sin\psi_k]^\top+\mathbf c_s(p_k,t),
\qquad
|\dot\psi_k|\leq \omega_k^{\max}.
\]

能耗不是距离的固定倍数，而是

\[
E_{k,s}(\pi)=
\int_{\pi}
\left(P_k^{\text{hotel}}+
P_k^{\text{prop}}(v_k,\mathbf c_s,\psi_k)\right)\,dt.
\]

### 2.2 原子覆盖需求

将 \(\Omega_f\) 离散为原子覆盖需求集合

\[
\mathcal U=\{u_1,\ldots,u_N\}.
\]

一个原子可以是小网格、测线段、传感器视场采样点或带时间权重的盲区巡逻点。每个原子具有

\[
d_u=(q_u,n_u,\Delta t_u,\mathcal K_u),
\]

其中 \(q_u\) 是重要度，\(n_u\) 是所需覆盖次数，\(\Delta t_u\) 是允许的重复访问间隔，\(\mathcal K_u\) 是能够执行它的无人艇集合。普通测绘任务通常 \(n_u=1\)；重点盲区可要求多次访问。

采用原子需求而不是直接令“大区域只属于一艘艇”，是因为它允许优化器在区域边界处进行跨艇交换，并用传感器足迹做最终覆盖验证。

若 \(n_u>1\) 且存在重访间隔，不只使用一个累计次数约束，而把需求展开为带允许时窗的副本

\[
\{u^{(1)},u^{(2)},\ldots,u^{(n_u)}\}.
\]

路线列只有在其实际访问时刻落入对应时窗时，才令 \(a_{u^{(r)}q}=1\)。这样可以防止优化器用短时间内连续扫过同一点来虚假满足周期巡逻要求。

### 2.3 多模式覆盖原语

对每个候选区域 \(R\) 和无人艇 \(k\)，生成模式集合

\[
\mathcal M_{R,k}
=
\{m=(g,\beta,e^{\text{in}},e^{\text{out}},v)\}.
\]

其中 \(g\) 是往复式、CFS 或特殊巡逻模式，\(\beta\) 是覆盖方向，\(e^{\text{in}},e^{\text{out}}\) 是进出端点，\(v\) 是模式内建议航速。每个模式记录：

- 覆盖的原子集合 \(A(m)\)；
- 各海流情景下的航时 \(T_{m,k}^s\) 与能耗 \(E_{m,k}^s\)；
- 转弯次数、累计转角和最小曲率半径；
- 覆盖重叠、边界漏扫风险与定位裕度；
- 起止位置、艏向与可连接安全时隙。

候选方向不只取最小几何宽度方向，而来自：

1. 区域主轴和主要障碍边方向；
2. 方向容差 BCD 给出的低转弯方向；
3. 海流 \(E-\beta\) 曲线的局部极小方向；
4. 与相邻区域连接代价较低的方向；
5. 用户规定的测线方向或法规约束方向。

每个方向至少生成 FF、FR、RF、RR 四种端点模式；不规则障碍密集区域同时生成 CFS 模式。优化器保留两类模式竞争，而不是提前用人工规则固定唯一分解。

### 2.4 路线列

对无人艇 \(k\)，一条路线列 \(q\in\mathcal Q_k\) 是

\[
q=(k,m_1,\gamma_{12},m_2,\ldots,m_h,\gamma_{h0}),
\]

即从基地出发，按某一顺序执行若干覆盖模式，经受障碍、动力学和海流约束的连接轨迹 \(\gamma\) 转移，并安全返航。路线列已经内嵌：

- 任务归属；
- 区域访问顺序；
- 覆盖方向和进出模式；
- 航速与必要等待；
- 续航、返航和运动学可行性。

路线属性包括 \(a_{uq}\)、\(T_q^s\)、\(E_q^s\)、\(N_q^{\text{turn}}\)、\(O_q^{\text{overlap}}\)、\(W_q^{\text{wait}}\) 等，其中 \(a_{uq}\) 是路线对原子 \(u\) 的覆盖次数。

这一步是本算法与“先分区，再做单艇 TSP”的根本区别：主问题选择的是完整多艇路线组合，任何一次跨艇任务交换都会连同顺序、模式、连接和冲突成本一起重新评价。

## 3. 词典序鲁棒优化目标

不采用一个难以解释的固定加权和直接混合所有指标。目标按优先级分三层求解。

### 第 0 层：硬可行性

必须满足：

\[
\sum_{k\in\mathcal K}\sum_{q\in\mathcal Q_k}a_{uq}x_q\geq n_u,
\quad \forall u\in\mathcal U,
\]

\[
E_q^s\leq B_k-R_k,
\quad \forall q\in\mathcal Q_k,\ \forall s\in\mathcal S,
\]

以及运动学、禁航区、返航、最小艇间距和必要的重复访问间隔约束。\(R_k\) 是应急返航电量。

### 第 1 层：最小化鲁棒最大完工时间

\[
Z_1=\min T_{\max},
\]

\[
T_{\max}\geq
\sum_{q\in\mathcal Q_k}T_q^s x_q,
\quad \forall k,s.
\]

该层直接优化舰队最晚完成者，避免一艘艇成为瓶颈。

### 第 2 层：在不显著牺牲 \(Z_1\) 的条件下最小化舰队能耗风险

\[
Z_2=
\min
\left[
\sum_s p_s\sum_k E_{k,s}
+\lambda_{\text{cvar}}
\operatorname{CVaR}_{\alpha}(E_{\text{fleet}})
\right],
\]

满足

\[
T_{\max}\leq (1+\varepsilon_1)Z_1^*.
\]

精确实验令 \(\varepsilon_1=0\)；工程大实例可设很小的容许值，例如 \(0.5\%\) 至 \(2\%\)，并公开灵敏度分析。

### 第 3 层：任务质量与可执行性

\[
\begin{aligned}
Z_3=\min\;&
\lambda_t\sum_q N_q^{\text{turn}}x_q
+\lambda_o\sum_u o_u
+\lambda_w\sum_q W_q^{\text{wait}}x_q\\
&+\lambda_f F_{\text{fragment}}
+\lambda_r R_{\text{navigation}},
\end{aligned}
\]

满足 \(Z_2\leq(1+\varepsilon_2)Z_2^*\)。其中

\[
o_u\geq
\sum_{k,q}a_{uq}x_q-n_u,\qquad o_u\geq0.
\]

\(F_{\text{fragment}}\) 惩罚同一艘艇任务过度碎片化，\(R_{\text{navigation}}\) 是狭窄水域、岸线误差和高流速区域的航行风险。第 3 层权重只在低优先级同层指标中使用，且所有指标先按物理上界归一化。

## 4. 模式扩展图与真实代价预言机

### 4.1 模式扩展图

建立有向图

\[
\mathcal G^M=(\mathcal V^M,\mathcal E^M).
\]

每个节点对应一个“区域—艇型—覆盖模式”，每条边连接前一模式的出口状态与后一模式的入口状态。不同艇的节点和边可不同，因此异构性不是事后修正，而是图结构的一部分。

### 4.2 双界连接代价

若直接为所有模式对计算连续动力学最优轨迹，预处理会过重。本文采用双界预言机：

1. 下界 \(c^-_{ij,k,s}\)：由 Eikonal/Fast Sweeping 距离场或放松转弯约束的最短时间计算，用于剪枝和下界；
2. 可行上界 \(c^+_{ij,k,s}\)：由带艏向状态的 Hybrid A*、状态格或等价的非完整约束搜索生成，再经 Dubins/直接配点平滑，用于构造实际路线；
3. 延迟精化：只有进入当前候选解、负约化成本列或冲突热点的边才计算高精度连续轨迹。

若论文需要报告“严格最优性差距”，上下界必须相对于同一个有限状态格模型定义；连续动力学平滑后的结果另行报告，不把离散模型最优误写成连续空间全局最优。

### 4.3 海流与能耗

每个覆盖模式和连接边都在情景集上积分航时与能耗。若预测场较大，先对相近情景聚类，再保留：

- 标称情景；
- 顺流优势情景；
- 逆流不利情景；
- 横流和局部涡流情景；
- 预测置信区间边界情景。

这样既不退化为只按距离计算，也避免情景数量无限增长。

## 5. 核心联合主问题

令二元变量 \(x_q=1\) 表示选择路线列 \(q\)，否则为 0。受限主问题的核心约束为：

\[
\sum_{q\in\mathcal Q_k}x_q=1,
\quad \forall k\in\mathcal K,
\]

\[
\sum_{k}\sum_{q\in\mathcal Q_k}a_{uq}x_q\geq n_u,
\quad \forall u\in\mathcal U,
\]

\[
\sum_{q\in\mathcal Q_k}T_q^s x_q\leq T_{\max},
\quad \forall k,s,
\]

\[
\sum_{q\in\mathcal Q_k}E_q^s x_q\leq B_k-R_k,
\quad \forall k,s.
\]

运动学连续性、单艇内部顺序和返航已经由路线列的定义保证。艇间冲突通过惰性冲突割加入。

若冲突预言机判定路线 \(q\) 与 \(r\) 在当前安全时隙集合中无法同时执行，则添加

\[
x_q+x_r\leq1.
\]

如果两条几何路线只需改变出发时刻或局部等待即可共存，则不永久禁止组合，而是生成带安全时隙和等待资源的新路线列。狭窄航道还可添加容量割：

\[
\sum_{q\in\mathcal Q(C,\tau)}x_q\leq \operatorname{cap}(C,\tau),
\]

其中 \(\mathcal Q(C,\tau)\) 是在时窗 \(\tau\) 占用水域资源 \(C\) 的路线集合。

这个模型统一处理四个过去被割裂的问题：

- 集合覆盖约束决定任务分配；
- 路线列的模式序列决定访问顺序；
- 模式节点决定覆盖方向和端点；
- 冲突割及安全时隙列决定多艇时空协调。

## 6. 精确引擎：分支—定价—割

### 6.1 为什么使用列生成

若对每个原子、艇、访问顺序、覆盖模式和时刻直接建立一个巨型 MIP，变量数量会随任务数指数增长。路线列模型仍然有指数多列，但只按需生成有价值的列，且天然适合舰队路线重组。

### 6.2 受限主问题

开始时只放入少量可行路线：

1. MCFS/MMRTC 生成的不规则区均衡路线；
2. BCO-ARA 或异构网格交易得到的能力均衡路线；
3. 最近插入和最小增量能耗路线；
4. 每个单艇的安全返航空路线。

先求 LP 松弛得到覆盖约束对偶价 \(\pi_u\)、单艇选路对偶价 \(\sigma_k\)、makespan 与资源约束对偶价。

### 6.3 定价子问题

对每艘艇 \(k\)，在模式扩展图上求资源受限基本最短路。路线 \(q\) 的约化成本一般形如

\[
\bar c_q=
c_q-\sigma_k-\sum_u\pi_u a_{uq}
-\sum_h\mu_h b_{hq},
\]

其中 \(b_{hq}\) 表示路线是否占用冲突资源 \(h\)。若存在 \(\bar c_q<0\) 的路线，则加入主问题。

定价状态至少包括：

\[
\ell=
(m_{\text{last}},t,E,\mathcal N_g,I_{\text{safe}},
\bar c,\text{covered-priority}),
\]

分别记录末模式、累计时间、能耗、ng 邻域记忆、安全时隙、约化成本和已覆盖重点需求。采用双向标签扩展与支配规则：

若两个标签到达同一末状态，且标签 A 在时间、能耗、风险、约化成本上均不劣，并覆盖不少于标签 B 的有效需求，则删除 B。

小中规模使用基本路径的精确定价；大规模根节点可使用 ng-route 松弛得到有效下界，并用启发式定价快速寻找负约化成本列。

### 6.4 分支策略

不优先对具体路线列分支，因为新列会破坏分支结构。采用：

1. 原子—艇归属分支：令原子 \(u\) 必须或禁止由艇 \(k\) 覆盖；
2. 模式相邻关系分支：强制或禁止模式 \(i\) 后接模式 \(j\)；
3. 必要时对狭窄航道先后关系分支。

这些约束可直接传递给定价子问题。

### 6.5 冲突检测与割平面循环

每次得到整数路线组合后执行连续时间冲突预言机：

1. 用艇体安全域对轨迹扫掠体做快速包围盒筛选；
2. 对候选冲突段做连续时间最近距离计算；
3. 检查交叉、追越、对遇和狭窄航道占用；
4. 若可通过等待消解，生成安全时隙路线列；
5. 若当前组合不可调度，添加冲突割并继续求解；
6. 直到所有艇对满足最小距离与航行规则。

与只在最终阶段用局部避碰不同，等待和绕行后的成本会返回主问题，因此优化器可以选择换艇、换序或换覆盖模式，而不是被迫修补一个已经固定的分配。

### 6.6 精确引擎的输出

在候选模式库、离散动力学模型和有限情景集固定时，若定价精确、分支完整且所有冲突割分离完毕，算法有限终止并返回该离散模型的全局最优解。时间截止时返回：

- 当前最好可行值 \(UB\)；
- 分支树或根节点松弛下界 \(LB\)；
- 相对最优性差距

\[
\operatorname{Gap}=
\frac{UB-LB}{\max(|UB|,\epsilon)}\times100\%.
\]

这使实验能够回答“当前算法离可证最优还有多远”。

### 6.7 独立下界与分层 Gap

为避免列生成尚未完全收敛时没有参照，同时计算三个相互独立的乐观下界。

第一，覆盖工作量下界。令 \(\tau_{uk}^-\) 为艇 \(k\) 完成原子 \(u\) 的放松覆盖时间，求连续分配松弛：

\[
\begin{aligned}
LB_{\text{work}}=\min\quad &L\\
\text{s.t.}\quad
&\sum_k y_{uk}\geq1,\quad \forall u,\\
&\sum_u \tau_{uk}^-y_{uk}\leq L,\quad \forall k,\\
&0\leq y_{uk}\leq1,\quad
y_{uk}=0\ \text{if}\ k\notin\mathcal K_u.
\end{aligned}
\]

它允许任务被分数分配且忽略连接、转弯和冲突，因此一定不高于真实最优 makespan。

第二，最远必要服务—返航下界：

\[
LB_{\text{return}}=
\max_{u\in\mathcal U}
\min_{k\in\mathcal K_u}
\left[
d_k^-(p_k^0,u)+
\tau_{uk}^-+
d_k^-(u,p_k^0)
\right].
\]

第三，完成精确定价或有效 ng-route 松弛后的根节点主问题下界 \(LB_{\text{CG}}\)。

最终一级目标使用

\[
LB_1=\max
\{LB_{\text{work}},LB_{\text{return}},LB_{\text{CG}}\}.
\]

若 \(LB_{\text{CG}}\) 尚未通过精确定价或可证明松弛收敛，则只能使用前两个严格下界，并将启发式列生成值标注为参考值。词典序目标应逐层报告：

- 一级 Gap：makespan 的 \(UB_1\) 与 \(LB_1\)；
- 二级 Gap：固定一级容许带后，能耗风险目标的上下界；
- 三级指标：通常报告可行值和相对改善，不在没有有效下界时声称最优。

## 7. 可扩展引擎：对偶引导的自适应大邻域搜索

大规模海域不要求分支—定价—割完整收敛，而使用与精确引擎共享成本和约束的 GLOBE-ALNS。

### 7.1 多源暖启动

并行概念上的初始解来源包括：

1. MCFS/MMRTC 路线；
2. 能力比例 BCO-ARA 分区后生成的路线；
3. 异构工作量网格交易路线；
4. 鲁棒最小增量插入路线；
5. 若当前论文已有方案可行，则把其输出无损转成路线列。

保留词典序目标最好的若干个不同结构解，而不是只保留单一初始解。

### 7.2 破坏算子

每次从当前解中删除一组模式或任务链：

- 瓶颈艇破坏：优先删除 \(T_{\max}\) 对应艇上的高边际任务；
- 高对偶价破坏：删除主问题中 \(\pi_u\) 高的难覆盖原子；
- 高遗憾破坏：删除“当前归属代价与次优艇代价差”大的任务；
- 冲突簇破坏：同时删除共享水域中相互冲突的任务链；
- 海流风险破坏：删除 CVaR 贡献高的逆流模式；
- 空间相关破坏：删除相邻的一组原子，允许区域边界整体迁移；
- 随机破坏：防止搜索长期锁定在同一分区拓扑。

### 7.3 修复算子

删除后的任务由以下操作重新插入：

- 跨艇 relocate；
- 两艇任务 swap；
- 两艇尾链 2-opt* 交换；
- 多任务 chain-exchange；
- 区域 split/merge；
- 覆盖方向变更；
- FF/FR/RF/RR 模式动态规划；
- Boustrophedon 与 CFS 模式替换；
- 共享航道重定时或访问次序互换。

插入代价不是原子到路线的欧氏距离，而是完整词典序边际向量：

\[
\Delta(u,k,q)=
(\Delta T_{\max}^{\text{rob}},
\Delta E^{\text{risk}},
\Delta N^{\text{turn}},
\Delta O^{\text{overlap}},
\Delta W^{\text{wait}},
\Delta R).
\]

先比较向量的第一项，只有相同或位于容许带内时才比较后续项。Regret-2/3 修复优先插入选择余地最少的任务。

### 7.4 对偶引导

每隔 \(I_{\text{dual}}\) 次迭代，求当前路线池主问题的 LP 松弛。对偶价用于：

- 识别覆盖困难、资源紧张或导致瓶颈的任务；
- 调整破坏算子的抽样概率；
- 引导新路线优先覆盖高对偶价原子；
- 识别被低估的航道冲突资源。

因此，对偶信息不是只用于精确求解，也成为大规模启发式的可解释搜索信号。

### 7.5 路线池重组

ALNS 每得到一个非支配或结构新颖的艇级路线，就加入路线池。每隔 \(I_{\text{pool}}\) 次迭代，在池内精确求解集合覆盖主问题。它可以把来自不同历史解的优质单艇路线重新组合，常能实现普通局部移动无法到达的跨解跃迁。

路径池大小通过以下规则控制：

- 总是保留当前最优解中的路线；
- 保留低约化成本和覆盖稀缺任务的路线；
- 删除长期未使用且与已有路线高度重复的列；
- 为每艘艇、每个主要区域拓扑保留最低数量的多样化路线。

### 7.6 接受与算子自适应

可行改进解始终接受；对轻微变差解采用温度或阈值接受以跳出局部最优，但任何第 0 层不可行解只可作为短暂修复状态，不能进入最终路线池。算子得分根据：

1. 是否刷新全局最好值；
2. 是否改善当前解；
3. 是否产生新结构；
4. 是否消除冲突或降低最优性差距；

进行指数平滑更新。

## 8. 自适应区域表示

单一 BCD 在细长障碍附近会产生许多狭窄子区；单一 CFS 在规则长条海域又可能牺牲测线规则性。GLOBE-USV 采用统一原子需求上的多表示候选库。

### 8.1 规则或测线约束区域

使用岸线简化、障碍膨胀、梯形/方向容差 BCD，将相邻且允许相近扫掠方向的单元合并为最大单调区域；生成多方向往复式模式。

### 8.2 不规则障碍密集区域

从距离场提取等距线并构造 isograph，生成 CFS 连续平滑模式；MMRTC 解只作为模式组合的初始建议。

### 8.3 统一选择

两类模式都映射到相同原子集合 \(\mathcal U\)。主问题以覆盖约束选择模式，因此不会因为表示来源不同而遗漏需求。若两个模式重复覆盖同一原子，重叠变量会计入代价。

为控制候选规模，采用：

- 覆盖集合完全相同且各项成本均被支配的模式删除；
- 起止状态相近时只保留若干 Pareto 模式；
- 对高对偶价和边界原子按需增加方向；
- 若最终覆盖验证发现漏扫，局部细分原子并重新生成模式。

## 9. 连续轨迹精修与双层安全执行

组合优化输出的是带艏向和时刻的可行骨架。随后进行直接配点或等价的连续轨迹优化：

\[
\min
\int
\left[
\eta_E P_k(t)+
\eta_\kappa \kappa_k(t)^2+
\eta_a a_k(t)^2
\right]dt
\]

并保持覆盖足迹、障碍安全距离、已确定的任务顺序和安全时隙不变。若精修使某项硬约束失效，轨迹不能被接受，必须返回组合层添加禁用边或冲突割。

执行时分两层：

- 慢层：GLOBE-USV 负责分钟至秒级的任务、路线和时刻联合重规划；
- 快层：规则约束的速度障碍、控制屏障函数或 DWA/RVO 类方法负责亚秒级动态避碰和 COLREGs 执行。

快层若产生的累计等待或偏航超过阈值，必须触发慢层，而不能长期让局部避碰悄悄破坏全局计划的质量。

## 10. 事件驱动的在线剩余任务重规划

### 10.1 状态机

借鉴故障容错论文的确定有限状态机思想，为每艘艇维护：

\[
\{\text{执行},\text{等待},\text{局部避碰},
\text{支援},\text{返航},\text{故障},\text{重规划}\}.
\]

状态机负责触发和安全降级，不直接替代全局优化。

### 10.2 触发条件

满足任一条件时触发重规划：

- 预测完工时间偏差超过阈值；
- 剩余能量的置信下界低于安全储备；
- 海流、风浪或禁航区变化使路线成本变化超过阈值；
- 无人艇故障、通信丢失或传感器幅宽改变；
- 局部避碰累计等待超过阈值；
- 新出现高优先级盲区需求；
- 路线间最小预测距离小于安全值。

### 10.3 剩余问题

重规划时：

1. 冻结所有艇已经执行的轨迹前缀；
2. 用实际传感器轨迹更新覆盖置信图；
3. 将尚未满足的原子及重复访问需求组成 \(\mathcal U^{\text{res}}\)；
4. 更新各艇位置、艏向、电量、故障状态和可用能力；
5. 删除不可达模式，复用仍有效的路线后缀和列；
6. 在滚动窗口 \(H\) 内求联合问题，并用下界估计窗口外剩余代价；
7. 先发布第一段经过冲突验证的可执行前缀，后台继续改善。

合作博弈价值只用于快速排序“哪艘艇最适合接管故障任务”；最终接管集合仍由联合主问题根据全舰队边际目标确定。

### 10.4 在线完整性条件

若自由水域有限、至少存在一艘可用艇能到达每个剩余原子、通信最终恢复、动态障碍不会永久封锁所有通路，且每次执行后真实更新覆盖集合，则反复求解剩余问题最终能够完成覆盖。若这些条件不满足，算法应输出“不可完成证书或原因”，而不是继续报告虚假的 100% 覆盖率。

## 11. 完整算法流程

### 算法 1：离线 GLOBE-USV

输入：任务多边形、障碍与安全域、海流情景、异构艇参数、覆盖要求、计算预算。

输出：每艘艇的带时间戳覆盖轨迹，目标值，覆盖验证结果，\(UB\)、\(LB\) 和 Gap。

1. 对地图做坐标统一、岸线简化、障碍膨胀和可达性检查。
2. 依据最小有效传感器分辨率生成原子覆盖需求 \(\mathcal U\)。
3. 生成方向容差 BCD 区域和 CFS 等距线区域候选。
4. 对每艘艇、每个区域生成多方向、四端点和多航速覆盖模式。
5. 用海流能耗模型计算模式的多情景成本，删除不可行或被支配模式。
6. 建立模式扩展图和双界连接代价预言机。
7. 用 MCFS/MMRTC、能力均衡分区、异构网格交易和贪婪插入生成多源初解。
8. 若实例规模位于精确预算内，运行词典序分支—定价—割：
   1. 求受限主问题；
   2. 求各艇定价子问题并加入负约化成本列；
   3. 对整数解做连续冲突检测并加入冲突割或安全时隙列；
   4. 分支并重复，直到证明最优或时间截止。
9. 否则运行 GLOBE-ALNS：
   1. 选择破坏和修复算子；
   2. 用完整边际代价跨艇重分配、重排和换模式；
   3. 检查能量、覆盖和时空冲突；
   4. 更新路线池与算子权重；
   5. 周期性求路线池主问题和 LP 对偶；
   6. 到时限后输出最好可行解与最强有效下界。
10. 对组合路径做连续轨迹精修。
11. 重新做传感器足迹覆盖验证、能量验证和连续艇间距验证。
12. 若验证失败，局部细分或添加禁用边/冲突割，并返回第 8 或第 9 步。
13. 输出轨迹、分层目标、下界、差距和完整验证报告。

### 算法 2：在线 GLOBE-RH

输入：离线路线池、实时艇状态、覆盖置信图、环境更新。

输出：经过安全验证的滚动执行前缀。

1. 状态机持续检测故障、偏差、冲突和新任务事件。
2. 无事件时沿已验证前缀执行，快层处理短时动态避碰。
3. 有事件时冻结已执行前缀并构造剩余原子集合。
4. 删除失效列，复用仍可行路线后缀，生成故障相关新模式。
5. 先用 ALNS 在短时限内得到可行解，再在后台用路线池主问题改进。
6. 若短时限内无可行解，执行安全等待或返航策略并报告原因。
7. 发布首段无冲突前缀，滚动更新，直至全部需求完成。

## 12. 可证明性质与声明边界

### 命题 1：覆盖可行性

如果所有被选路线均按定义可执行，且主问题满足

\[
\sum_{k,q}a_{uq}x_q\geq n_u,\quad\forall u,
\]

则离散原子模型中的全部覆盖需求都被满足。连续区域覆盖仍需用传感器足迹扫掠体进行后验验证；原子分辨率越细，离散误差越小。

### 命题 2：联合模型不劣于被纳入候选集的顺序式基线

若某个“先分区—后单艇 TSP—再避碰”基线的最终可行多艇路线被完整加入路线池，并使用相同词典序目标评价，则 GLOBE-USV 主问题的最优解不可能劣于该基线。原因是基线路线组合本身就是联合模型的一个可行解。

这是一条很重要的实验设计规则：把现有论文算法输出加入初始路线池，联合算法至少具有保底解，改进来自更大的联合搜索空间。

### 命题 3：有限模型最优性

当模式库有限、动力学被有限状态格表示、情景集有限、定价精确、分支规则完备且冲突割能够分离所有不兼容整数解时，分支—定价—割有限终止并返回该有限模型的全局最优解。

### 命题 4：任意时刻质量证书

只要有一个可行整数解和一个有效松弛下界，就可随时计算 Gap。算法提前终止不再只能说“收敛到了较好解”，而能报告例如“在当前模型下距全局最优不超过 3.2%”。

### 声明边界

以下表述不能在实验前写进论文结论：

- “算法获得连续海洋环境下的全局最优解”；
- “算法必然优于所有现有方法”；
- “大规模 ALNS 的解具有固定近似比”；
- “有限仿真足以证明真实海况鲁棒性”。

可严谨声明的是：

- 精确引擎对明确给定的有限候选/离散模型可证明最优或给出差距；
- 大规模引擎给出经过验证的可行解，并相对于有效下界报告经验差距；
- 连续轨迹和真实海况性能由独立验证与湖/海试验支持。

## 13. 面向 SCI 一区、顶刊或顶会的实验设计

算法设计只有与下列验证体系配套，才有机会支撑高水平投稿。

### 13.1 研究问题

- RQ1：联合优化是否显著优于“分区 + 单艇 TSP + 局部避碰”？
- RQ2：改进来自任务重分配、访问顺序、覆盖模式还是冲突反馈？
- RQ3：在小实例上是否达到最优；在中大实例上 Gap 有多大？
- RQ4：海流、异构性和动态障碍增强时，性能是否稳定？
- RQ5：故障后能否在有限时间内恢复覆盖且保持返航安全？
- RQ6：算法时间和内存如何随原子数、艇数、模式数和障碍密度增长？

### 13.2 数据集分层

1. 可精确求解小实例：\(N=10\) 至 \(40\)，\(K=2\) 至 \(4\)，用于穷举/完整 MIP/完整分支—定价—割对照。
2. 中等实例：\(N=50\) 至 \(300\)，\(K=3\) 至 \(12\)，要求报告 \(UB\)、\(LB\)、Gap 与时间曲线。
3. 大规模实例：\(N=500\) 至 \(5000+\)，\(K=5\) 至 \(100\)，比较固定时间预算下的最好可行值和稳定性。
4. 地图类别：规则海湾、狭长水道、岛群、障碍密集区、非凸岸线和多个窄通道。
5. 异构类别：不同初始点、航速、转弯半径、传感器幅宽、电量和任务能力。
6. 环境类别：无流、均匀流、空间变化流、时间变化流、预测偏差和极端不利情景。
7. 在线类别：单艇故障、传感器降级、通信延迟、临时禁航区和移动目标。

### 13.3 必须对比的基线

本地海洋方法：

- force allocation + improved branch-and-bound；
- BCO-ARA + GDRA/APF；
- adaptive Voronoi + GTS-STC + cooperative game；
- DASP + fused A* + DQN/DWA-APF；
- FSM distance field + Hungarian + IHO；
- heterogeneous grid trading + backbone tree evolution。

近三年高水平 MCPP 方法：

- MIP-MMRTC；
- LS-MCPP；
- MCFS；
- TMSTC*；
- T-RO 2025 的 ESTC/LS-MCPP + path deconfliction；
- ICRA 2025 的 NSGA-II MCPP。

关键内部基线：

- Sequential-GLOBE：使用相同覆盖模式和真实成本，但先固定分配再逐艇优化；
- Joint-no-bound：使用相同联合邻域，但不求 LP/定价下界；
- Oracle-cost versus Euclidean-cost：检验真实连接代价的价值。

只有使用相同地图、相同安全域、相同覆盖分辨率、相同返航要求和相同计算预算，比较才公平。

### 13.4 评价指标

主指标：

- 鲁棒 makespan；
- 期望总能耗与能耗 CVaR；
- 最优性差距 Gap；
- 完整覆盖率和重复覆盖率；
- 最小艇间距离及冲突数量；
- 算法首次可行解时间、达到 1%、3%、5% Gap 的时间。

次指标：

- 总航程、总航时、等待时间；
- 转弯次数、累计转角、曲率峰值；
- 各艇完工时间变异系数；
- 应急返航电量；
- 在线重规划时间、任务恢复率和故障后额外成本；
- 真实轨迹跟踪误差与通信负载。

### 13.5 消融实验

至少移除以下模块逐项验证：

1. 去掉任务—顺序联合，只保留固定分区；
2. 去掉覆盖模式选择，只保留一个最小宽度方向；
3. 去掉海流能耗模型，只按距离；
4. 去掉冲突反馈，只做最终局部避碰；
5. 去掉对偶引导；
6. 去掉路线池重组；
7. 去掉 CFS 或去掉方向容差 BCD；
8. 去掉鲁棒情景/CVaR；
9. 去掉在线列复用；
10. 固定加权和替代词典序目标。

最关键的消融是第 1 项，它直接证明论文新增的“全局联合优化”是否真正贡献了改进。

### 13.6 统计规范

- 随机算法每个实例至少独立运行 30 次；
- 同一实例使用配对随机种子和相同墙钟时间；
- 报告中位数、四分位距、均值和 95% bootstrap 置信区间；
- 两算法比较用配对 Wilcoxon 检验，多重比较用 Holm 校正；
- 同时报告效应量，而不是只报告 \(p\) 值；
- 公开失败实例、内存超限和无可行解次数；
- anytime 算法绘制解质量—时间和 Gap—时间曲线。

### 13.7 精确性验证

小实例必须做三重交叉验证：

1. 独立枚举或紧凑 MIP；
2. 完整分支—定价—割；
3. GLOBE-ALNS。

若前两者目标一致，则验证精确引擎；ALNS 与最优值的差异给出经验 Gap。中实例至少报告根节点列生成下界和最终上界。大实例若只能使用松弛下界，必须明确标注“相对松弛下界差距”，不能称为精确最优性差距。

### 13.8 物理实验

建议至少使用 3 至 6 艘异构 USV，在湖泊或封闭近岸水域完成：

- 无流/弱流静态覆盖；
- 不同传感器幅宽与电量；
- 狭窄通道会遇；
- 人工触发一艘艇故障；
- 临时动态障碍和局部禁航区；
- 同一任务由顺序式基线与 GLOBE-USV 交叉重复。

物理实验不是只展示轨迹图，还应同步记录电流、电压、定位、速度、控制输入、覆盖足迹和实际完成时间。

## 14. 预期投稿贡献的凝练方式

在完成实验后，可将贡献凝练为以下四点：

1. 提出一种异构多 USV 多模式路线列模型，首次在本文所研究的任务设定中统一任务分配、访问顺序、覆盖模式、海流能耗和时空冲突。
2. 提出精确与可扩展共享的双引擎求解框架：分支—定价—割提供有限模型最优性证书，对偶引导 ALNS 与路线池重组处理大规模实例。
3. 提出覆盖—冲突闭环和事件驱动剩余任务重规划，使等待、绕行与故障代价能够反馈到舰队级决策。
4. 建立从精确小实例、最优性差距、统计消融到多艇湖试的验证体系，明确区分局部模块提升、经验近优与可证最优。

“首次”一词只能在投稿前完成更系统的数据库检索后保留。当前文档可称为“拟创新点”，不能仅凭已有有限文献直接作绝对新颖性声明。

真正需要作为本文原创重点进行理论和实验论证的是：

1. 多模式异构路线列对任务—顺序—模式—时刻的统一表达；
2. 词典序鲁棒集合覆盖主问题；
3. 覆盖路线分支—定价与连续冲突割的闭环；
4. 对偶引导 ALNS 和跨历史路线池重组；
5. 精确引擎、可扩展引擎和在线剩余任务共享同一成本/约束接口。

上述五点不是任意模块堆叠：统一路线列是核心数据结构，主问题负责舰队级选择，精确与启发式引擎只是同一模型的两种求解方式，在线算法则是在相同模型上冻结已执行前缀后重新求解。

## 16. 实施前的算法审计清单

虽然本文不写代码，但未来实现必须逐项通过以下审计：

- 是否真正允许任务跨艇移动，而不是只在固定分区内优化？
- 每次跨艇移动后是否重新计算连接、海流能耗、转弯和冲突等待？
- 是否以最终可执行轨迹而不是代理距离计算 makespan？
- 是否有独立传感器足迹验证 100% 覆盖？
- 是否有连续时间艇间距验证？
- 所报告下界是否对当前模型有效？
- Gap 是相对严格下界、松弛下界还是仅相对最好已知值？
- 离散最优与连续空间最优是否明确区分？
- 基线是否使用相同约束、地图分辨率和计算预算？
- 随机实验是否报告分布、显著性和效应量？
- 故障后是否从真实剩余覆盖集合重规划？
- 局部避碰产生的代价是否会反馈到全局层？

只要其中前三项未满足，算法仍然可能只是“多个单艇 TSP 的组合”；只要第六至第八项未满足，实验仍然不能证明当前结果是较优解。

## 17. 结论

GLOBE-USV 的研究主线应明确为：

\[
\text{覆盖原子与多模式}
\rightarrow
\text{异构艇完整路线列}
\rightarrow
\text{舰队词典序联合优化}
\rightarrow
\text{连续冲突闭环}
\rightarrow
\text{可证下界与在线重规划}.
\]

这条主线直接修复当前论文“只优化单无人艇旅行商问题，无法保证舰队总体较优”的结构性缺陷。其学术价值不在于把更多元启发式名称串联起来，而在于用一个统一模型把过去割裂的决定重新耦合，并通过可行上界、有效下界、最优性差距和严格实验回答多无人艇方案究竟有多好。

## 附录 B：外部新增文献的使用约束

外部检索得到的论文只在满足以下条件时用于支撑核心设计：

1. 发表时间位于 2023 年 7 月至 2026 年 7 月检索窗口；
2. 来源为 IEEE Transactions on Robotics、IEEE Robotics and Automation Letters、Artificial Intelligence，或 AAAI、ICAPS、ICRA 等高水平期刊/会议；
3. 技术信息优先以出版社页面、会议官方页面、作者公开论文或 DOI 页面核验；
4. 只引用论文实际提出的算法，不把本文推导出的扩展错误归到原作者名下；
5. Ocean Engineering 等本地论文主要作为海洋应用证据和基线，不用期刊分区标签代替算法质量论证。

因此，GLOBE-USV 中的分支—定价路线列、对偶引导 ALNS、词典序鲁棒主问题和冲突割反馈是基于多篇工作的再抽象与新增组合结构，不能写成任何一篇既有论文算法的简单改名。
