# CROWN-BPC 最小精确实现与首个可证实验

## 1. 已实现的范围

当前代码实现了 CROWN-MCPP 的“小规模、有限模式、有限时间网格”精确内核，目标是先回答一个可被严格检验的问题：

> 在使用相同路线候选、相同冲突资源和精确后处理调度器时，允许冲突代价反馈到任务/路线/模式选择，是否能严格优于固定路线后的顺序式消冲突？

实现包含两套相互独立的求解路径：

1. `CROWN-ENUM`：枚举任务精确覆盖的路线组合；对每个组合枚举共享资源的全部先后关系，用 DAG 最长路得到最小 makespan；
2. `CROWN-BPC`：在完整有限时间扩展列集合上执行受限主问题、精确定价、兼容分支和惰性资源容量割。

二者只共享数据模型，不共享优化过程。小实例必须满足：

\[
Z_{\mathrm{BPC}}=Z_{\mathrm{ENUM}},\qquad LB=UB,\qquad Gap=0.
\]

## 2. 为什么顺序基线是“强基线”

`Sequential-ExactPost` 并未使用简单的先到先服务规则。它执行：

1. 固定任务分配；
2. 每个智能体独立选择名义时间最短的路线；
3. 冻结任务、顺序和模式；
4. 使用与联合 oracle 相同的精确共享资源调度器，枚举所有冲突先后关系。

因此，联合算法的优势不能归因于“后处理器太弱”。差距只来自顺序方法无法让冲突等待反向改变任务归属、访问顺序或覆盖模式。

## 3. 构造性严格改进实例

对每个智能体设置一个必须完成的任务，并提供两条路线：

- 公共路线：耗时 1，占用同一个容量为 1 的共享航道；
- 私有路线：耗时 \(1+\epsilon\)，不占用共享航道。

其中 \(0<\epsilon<K-1\)。顺序方法为每个智能体独立选择名义更短的公共路线。即使随后进行精确消冲突，\(K\) 条公共路线也必须串行：

\[
T_{\mathrm{seq}}=K.
\]

联合方法选择至多一条公共路线，其余使用私有路线，所有路线可同时开始：

\[
T_{\mathrm{joint}}=1+\epsilon.
\]

故严格改进和比值为：

\[
T_{\mathrm{seq}}-T_{\mathrm{joint}}=K-1-\epsilon>0,
\qquad
\frac{T_{\mathrm{seq}}}{T_{\mathrm{joint}}}
=\frac{K}{1+\epsilon}.
\]

当 \(\epsilon\) 为固定常数时，该比值随智能体数量线性增长。这证明冲突无感的顺序规划不存在与 \(K\) 无关的常数近似保证。

## 4. CROWN-BPC 的精确性闭环

当前 BPC 执行严格的两阶段词典序优化：

1. 第一阶段最小化 makespan；
2. 固定第一阶段最优 makespan，第二阶段最小化路线能耗与等待能耗。

受限主问题 LP 由项目内置的两阶段单纯形求解，不依赖 SciPy、Gurobi、CPLEX 或 OR-Tools。定价对每个智能体扫描完整的有限时间扩展路线集合，加入约化成本最负的列；无负约化成本列时，当前 LP 是该分支节点完整列集合上的最优松弛。

分支依次使用“任务—智能体”、单元相邻和容量一狭窄资源的首次占用先后决定；无法形成这些结构分支时，才使用完备的时间路线变量后备分支。整数解出现后分离尚未激活的资源—时间容量冲突。新增冲突割后，代码会重新求 LP 并重新定价；先后分支行及冲突容量行的对偶价格均进入标签定价，能够影响尚未生成的路线。

## 5. 运行方式

在项目根目录运行：

```bash
python examples/run_crown_minimal_exact.py --agents 3 --epsilon 0.5
```

该配置的确定性结果为：

| 方法 | makespan | 总能耗 |
|---|---:|---:|
| Sequential-ExactPost | 3.0 | 3.0 |
| CROWN-ENUM | 1.5 | 4.0 |
| CROWN-BPC | 1.5 | 4.0 |

CROWN-BPC 同时报告 makespan 的 `LB=1.5`、`UB=1.5`、`Gap=0`，以及第二阶段能耗的 `energy_LB=energy_UB=4.0`、`energy_Gap=0`，并报告生成列数、定价轮数、分支节点数、冲突分离轮数。

测试命令：

```bash
python -m unittest -v tests.test_crown_mcpp
```

## 7. 结论边界

当前可以严格声称的是：

- 相对于给定有限路线/模式库，`CROWN-ENUM` 是连续时间精确解；
- 相对于给定时间步长、时间上界及由此产生的完整有限时间扩展列集合，`CROWN-BPC` 是词典序精确解；
- 构造实例严格证明联合冲突反馈优于固定路线的精确后处理；
- 整数时间小实例上，BPC 已与独立枚举 oracle 逐例交叉校验。

当前不能声称的是：

- 对未经离散的任意连续空间问题全局最优；
- 对尚未进入候选库的扫掠角、连接轨迹或责任单元拓扑全局最优；
- 已具备大规模可用性。时间扩展和精确定价均为指数级，超出列上限会明确失败，不会静默截断；
- 已完成 CROWN-LNS。它仍是后续规模化阶段，而不是本次“最小精确版”的一部分。

现有几何规划器没有被替换。它可以继续生成覆盖模式和连接段，再通过 `CrownSegmentRouteCandidate`、`build_crown_instance_from_segment_candidates` 接入精确内核；这样保留当前运动学/障碍验证，同时把最终路线组合交给联合优化。
