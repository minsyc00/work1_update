# 200 和 400 地图生成要求

## 目标

生成 `200m x 200m` 和 `400m x 400m` 两张静态障碍地图，用于当前 USV 多艇覆盖路径规划算法测试。地图必须具备多种障碍形态，但不能为了复杂性制造大量碎片区域、窄通道、深凹死胡同或长距离屏障，核心目标是让当前算法能够完整跑完并生成结果图。

## 必须包含的障碍类型

两张地图都必须包含以下障碍类型：

- 圆形障碍：`circle`
- 椭圆障碍：`ellipse`
- 凸多边形障碍：`polygon`，例如三角形、梯形、六边形或不规则凸岛
- 凹多边形障碍：`polygon`，但必须是浅凹、宽凹口，不允许 U 形深凹或迷宫式凹陷
- 类似 S 型障碍：优先用 2 到 3 段不重叠的椭圆或短凸多边形链表达，不建议使用一个长凹多边形直接切开地图

## 当前算法适配原则

- 障碍物之间必须留出宽通道，避免障碍膨胀后相交或形成极窄缝隙。
- 不使用密集小礁石簇；小障碍数量要少，且不能沿同一条线密集排列。
- 凹多边形只能用于形态测试，不得形成深口袋、长走廊或需要 residual planner 大量补救的区域。
- S 型障碍应短而平缓，不能横向或纵向贯穿大半张地图。
- 地图应保留大块矩形、梯形或近凸自由水域，便于当前大块凸区域分解和简单往复扫描。
- 障碍物不要贴边，边界附近保留足够空间用于 U-turn、connector 和 residual backfill。

## 数量与间距建议

### 200x200 地图

- 障碍总数建议：`8 到 10` 个。
- 推荐 USV 数量：`4`。
- 原始障碍之间最小距离建议不小于 `12m`。
- 膨胀障碍之间最小距离建议不小于 `8m`。
- 凹多边形凹口宽度建议不小于 `24m`，凹入深度不超过障碍短边的 `35%`。
- S 型障碍建议占用尺度不超过 `50m x 70m`。
- 大块区域分解后目标区域数建议不超过 `55`。

### 400x400 地图

- 障碍总数建议：`10 到 14` 个。
- 推荐 USV 数量：`5 或 6`。
- 原始障碍之间最小距离建议不小于 `20m`。
- 膨胀障碍之间最小距离建议不小于 `12m`。
- 凹多边形凹口宽度建议不小于 `40m`，凹入深度不超过障碍短边的 `35%`。
- S 型障碍建议用 3 段短障碍组成，整体尺度不超过 `90m x 140m`，段与段之间膨胀后仍保持可通行间距。
- 大块区域分解后目标区域数建议不超过 `70`。

## 形态设计要求

- 圆形和椭圆障碍用于模拟礁石、浮标或浅滩，数量不宜过多。
- 凸多边形用于模拟岛屿、平台或浅滩边界，应优先选三角形、梯形、六边形或不规则凸多边形。
- 凹多边形只允许浅凹，例如轻微 L 型或带一个宽缺口的多边形；不允许 U 型深口袋。
- S 型障碍不应是单个长曲折多边形；推荐使用多段椭圆链，让视觉上呈 S 型，但算法上仍接近多个凸障碍。
- 障碍布局应分散在不同象限，避免集中在地图中央造成区域连接瓶颈。

## 静态检查要求

生成地图后必须通过以下检查：

- JSON 语法有效。
- `load_map_json` 可读取。
- `load_map_for_planner` 可解析。
- 障碍类型均为当前 loader 支持类型：`rectangle`、`ellipse`、`circle`、`polygon`。
- 所有障碍坐标在任务边界内。
- 原始障碍之间无重叠。
- 按当前安全距离和足迹宽度膨胀后，障碍之间仍无重叠。
- 大块区域分解能够启用，且区域数量控制在上述建议范围内。

## 运行验收要求

地图生成后应先做分阶段验证，再做完整实验：

```bash
python3 examples/run_paper_style_region_tsp_experiment.py \
  --map maps/static_obstacle_map_200x200_mixed_obstacles/static_obstacle_map_200x200_mixed_obstacles.json \
  --rmin 2 --target-coverage 0.99 --no-render --monitor-stages
```

```bash
python3 examples/run_paper_style_region_tsp_experiment.py \
  --map maps/static_obstacle_map_400x400_mixed_obstacles/static_obstacle_map_400x400_mixed_obstacles.json \
  --rmin 2 --target-coverage 0.99 --no-render --monitor-stages
```

验收时重点关注：

- `coverage_fraction >= 0.99`
- `skipped_region_count = 0` 或 skipped region 有明确、合理原因
- `out_of_bounds_segment_count = 0`
- `obstacle_collision_segment_count = 0`
- `kinematic_infeasible_segment_count = 0`
- `coverage_pattern_generation`、`build_sweep_paths`、`agent_region_tsp` 不出现明显长时间卡死
- 生成结果图时，`04_selected_region_sweep_patterns.png` 和 `08_final_region_tsp_coverage_path.png` 能正常输出

## 设计取舍

本地图不是为了最大化障碍复杂度，而是为了在当前算法能力范围内测试多形态障碍。复杂性优先级为：

1. 保证算法完整跑完。
2. 保证约束检查通过。
3. 保证覆盖率达标。
4. 保留圆、椭圆、凸多边形、凹多边形和 S 型障碍。
5. 再考虑提高障碍密度和形态复杂度。
