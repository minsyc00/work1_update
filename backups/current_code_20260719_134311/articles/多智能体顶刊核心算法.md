# CROWN-MCPP：带可证界的异构多智能体多模式覆盖与路径消冲突联合优化

> 英文名称：Conflict-aware Route-column Optimization With certifiable bouNds for Heterogeneous Multi-Robot Coverage Path Planning。
>
> 算法简称：CROWN-MCPP。
>
> 文档性质：面向多智能体顶刊顶会的核心算法稿，不包含实现代码。
>
> 与原方案关系：保留 GLOBE-USV 中最有价值的多模式路线列、冲突反馈、精确/可扩展双引擎和可证上下界；删除在线故障状态机、CVaR、多种分解自由切换、动态目标和过多外围模块，把理论与实验资源集中到一个可验证的核心贡献。

## 摘要

现有多机器人覆盖路径规划通常先将区域分配给各机器人，再分别优化单机器人覆盖路径，最后用多智能体路径规划或局部避碰消除冲突。这种顺序式结构存在系统性缺陷：分配阶段看不到覆盖方向、非完整运动学、连接轨迹和冲突等待的真实代价；路径消冲突又可能显著改变原有 makespan，使前序分配失去最优性。

本文提出 CROWN-MCPP，一种面向异构非完整多智能体的冲突感知多模式路线列联合优化算法。算法首先将自由空间划分为互不重叠的覆盖单元，为每个“单元—机器人”组合生成包含扫掠方向、进出位姿和完整覆盖轨迹的有限模式集。随后在时间扩展的模式图上定义一条机器人完整路线列，使其同时编码任务归属、访问顺序、覆盖模式、连接轨迹、能量消耗和时空资源占用。主问题以严格词典序最小化舰队 makespan 和总能耗；定价子问题为带元素性、能量和安全时隙资源的最短路。算法采用惰性冲突资源分离：每个连续轨迹冲突被映射为有限状态—时间图中的容量资源，新增割不仅排除当前冲突组合，其对偶价格还反馈到后续路线生成。

小中规模实例由分支—定价—割求解，在有限模式与状态—时间模型上给出全局词典序最优解或严格最优性差距。大规模实例采用对偶引导的路线大邻域搜索，并周期性求解路线池集合划分问题；当根节点精确定价完成时，启发式解仍具有有效 anytime 质量证书。本文进一步证明冲突无感顺序规划的近似比可随机器人数量线性恶化，并给出覆盖与安全裕度定理、路线列主问题等价定理、惰性冲突分离有限完备性定理、词典序全局最优性定理、候选角度离散误差界以及服务工作量下界定理。

本文的目标不是组合多个现有启发式模块，而是建立一个单一、闭合且可证明的联合优化内核：覆盖任务、运动模式和多智能体冲突由同一列生成过程共同决定。

## 1. 顶刊版算法的取舍

### 1.1 保留的核心

从原 GLOBE-USV 中保留四项：

1. 多模式路线列：一条列同时编码机器人、覆盖任务、访问顺序、覆盖方向和时间安排；
2. 冲突反馈：消冲突不是最终后处理，冲突资源价格进入定价子问题；
3. 精确与可扩展双引擎：分支—定价—割负责证明，大邻域搜索负责规模；
4. 可证上下界：任何算法结果都尽可能同时报告 \(UB\)、\(LB\) 和 Gap。

### 1.2 主动删除或降级的模块

| 原方案内容 | 顶刊版处理 | 取舍原因 |
|---|---|---|
| 在线故障状态机、合作博弈接管 | 删除，作为后续论文 | 与离线联合优化是第二个完整研究问题，会削弱主线 |
| 多情景 CVaR 鲁棒优化 | 降为讨论项 | 会显著扩大定价状态，先证明确定性时变模型 |
| BCD、CFS、任意分解之间自由切换 | 固定单元分解，仅把 BCD/CFS 作为单元内部候选模式 | 保证责任单元互不重叠，使主问题、下界和证明闭合 |
| DQN、APF、DWA、RVO 等局部避碰 | 不属于核心算法 | 论文直接规划无冲突时间化路径，不用局部层掩盖规划冲突 |
| 大量低层指标的加权和 | 删除 | 只保留严格词典序的 makespan 与总能耗 |
| 区域 split/merge 在线搜索 | 删除 | 固定原子单元后，精确模型与基线比较更公平 |
| 动态未知障碍和通信失效 | 删除 | 本文限定已知环境；动态在线协同另行研究 |

### 1.3 明确的问题边界

CROWN-MCPP 解决：

- 已知二维自由空间中的完整覆盖；
- 多个异构非完整机器人；
- 不同起终点、航速、转弯半径、传感器幅宽与电量；
- 已知的静态或确定性时变运动代价，例如预测海流；
- 机器人之间的连续时间安全间隔；
- cover-and-return 或指定终点任务；
- 词典序最小 makespan，再最小总能耗。

CROWN-MCPP 当前不解决：

- 未知环境探索；
- 随机动态障碍；
- 故障恢复；
- 分布式通信协商；
- 连续空间中不经离散的普适全局最优。

主动限定范围不是降低算法水平，而是使核心贡献能够被严格定义、证明和复现。

## 2. 研究定位与现有工作的实质差异

近三年高水平 MCPP 工作给出了明确的审稿标尺。

- [Mixed Integer Programming for Time-Optimal Multi-Robot Coverage Path Planning with Efficient Heuristics](https://arxiv.org/abs/2306.17609)，IEEE Robotics and Automation Letters 2023：以 MMRTC 和 MIP 给出时间最优树覆盖与有界性质。CROWN-MCPP 不优化覆盖树，而优化带模式、时间和冲突资源的完整机器人路线。
- [Large-Scale Multi-Robot Coverage Path Planning via Local Search](https://ojs.aaai.org/index.php/AAAI/article/view/29707)，AAAI 2024：用 ESTC 和路径级局部搜索直接改善 makespan。CROWN-MCPP 借鉴“直接优化最终路径”的原则，但增加异构模式选择、列生成下界和联合冲突资源。
- [Multi-Robot Connected Fermat Spiral Coverage](https://ojs.aaai.org/index.php/ICAPS/article/view/31520)，ICAPS 2024：以等距线图、CFS 和 MMRTC 处理不规则障碍区。CROWN-MCPP 将 CFS 限定为单元内部候选模式，不让其固定最终任务分配。
- [Large-Scale Multirobot Coverage Path Planning on Grids With Path Deconfliction](https://www.ieee-ras.org/news/large-scale-multirobot-coverage-path-planning-on-grids-with-path-deconfliction/)，IEEE Transactions on Robotics 2025：把 ESTC、局部搜索与 MAPF 消冲突结合，并验证百机器人规模。CROWN-MCPP 的关键差异是冲突资源位于主问题与定价问题内部，冲突等待可以改变任务归属和覆盖模式，而不只是对既定覆盖路径后处理。
- [An Efficient NSGA-II-Based Algorithm for Multi-Robot Coverage Path Planning](https://doi.org/10.1109/ICRA55743.2025.11128792)，ICRA 2025：使用 donation mutation、multiple-parent crossover 和精英路径库搜索多目标覆盖路径。CROWN-MCPP 保留跨机器人路线交换和路径池思想，但给出精确主问题、下界和词典序证书。

本地 USV 论文的共同局限是把任务分配、单艇 TSP、连续轨迹和平面避碰顺序求解。CROWN-MCPP 的论文级研究问题可凝练为：

> 能否在同一个有限但可扩展的模型中，联合选择异构机器人的覆盖责任、单元顺序、覆盖模式和无冲突时刻表，并同时获得可行解与可验证的最优性差距？

### 2.1 核心创新声明

论文应只主张以下三项核心贡献：

1. 冲突感知多模式路线列模型：把覆盖分配、顺序、进出模式和时空资源统一到一条完整路线列；
2. 覆盖—冲突联合分支—定价—割：连续冲突经保守时空管映射为容量资源，其对偶价格反向影响新列；
3. 共享对偶与路线池的可扩展求解器：精确根松弛、路线大邻域搜索和池内整数重组共享同一数学模型，并输出 anytime 证书。

覆盖模式生成、海流代价、Hybrid A*、CFS 和轨迹平滑是支撑模块，不单独宣称为创新。

## 3. 问题定义

### 3.1 环境与机器人

自由空间为

\[
\Omega_f=\Omega\setminus
\left(\mathcal O\oplus B(r_{\mathrm{obs}})\right),
\]

其中 \(\Omega\) 为任务区域，\(\mathcal O\) 为静态障碍，\(r_{\mathrm{obs}}\) 包含机器人外形和静态定位裕度。

机器人集合为 \(\mathcal K=\{1,\ldots,K\}\)。机器人 \(k\) 的参数为

\[
\Theta_k=
\left(
p_k^0,\psi_k^0,p_k^g,
v_k^{\min},v_k^{\max},
\omega_k^{\max},\rho_k,
w_k,B_k,P_k
\right).
\]

它们分别表示初始位姿、终点、速度范围、最大转艏率、最小转弯半径、有效覆盖幅宽、电量和功率模型。USV 的平面运动学写为

\[
\dot p_k=
v_k
\begin{bmatrix}
\cos\psi_k\\
\sin\psi_k
\end{bmatrix}
+c(p_k,t),
\qquad
|\dot\psi_k|\leq\omega_k^{\max},
\]

其中 \(c(p,t)\) 是已知或预测的确定性时变海流。对地面机器人令 \(c=0\)。

### 3.2 固定责任单元

将 \(\Omega_f\) 分解成闭包并集等于任务区域、内部互不重叠的单元集合

\[
\mathcal I=\{1,\ldots,n\},
\qquad
\bigcup_{i\in\mathcal I}C_i=\Omega_f,
\qquad
\operatorname{int}(C_i)\cap
\operatorname{int}(C_j)=\varnothing.
\]

分解采用方向容差 BCD 或最大单调单元合并。单元可以很小，但在核心模型中不再改变拓扑。每个单元必须且只需由一艘机器人负责一次；边界上的安全重叠不视为重复任务。

固定责任单元有三个作用：

1. 使“完整覆盖”转化为精确一次责任约束；
2. 使服务工作量下界具有可加性；
3. 使精确求解的最优对象清楚：结论相对于给定单元与模式库成立。

### 3.3 单元多模式

机器人 \(k\) 覆盖单元 \(i\) 的可行模式集合为

\[
\mathcal M_{ik}=
\{m_{ik}^{1},\ldots,m_{ik}^{q_{ik}}\}.
\]

一个模式

\[
m=(\gamma_m,z_m^{\mathrm{in}},z_m^{\mathrm{out}},
\tau_m,e_m,\mathcal T_m)
\]

包含完整单元内覆盖轨迹 \(\gamma_m\)、入口与出口位姿、服务时间、能耗和保守时空占用管 \(\mathcal T_m\)。

候选模式只包括：

1. 若干离散扫掠角的 Boustrophedon 模式；
2. 每个角度的 FF、FR、RF、RR 端点方向；
3. 对确实不适合往复式覆盖的单元，可加入一个或少量 CFS 模式。

模式必须单独通过：

- 传感器足迹完整覆盖验证；
- 最小转弯半径验证；
- 静态障碍安全验证；
- 能量和最大服务时间验证。

无效模式不进入全局优化。

### 3.4 连接运动原语

从模式 \(m\) 的出口到模式 \(m'\) 的入口，机器人 \(k\) 使用有限状态—时间格上的连接原语

\[
g=(m,m',t^{\mathrm{dep}},\gamma_g,\tau_g,e_g,\mathcal T_g).
\]

连接由带艏向的状态格或 Hybrid A* 产生。每条连接的保守时空管覆盖在整个时间段内机器人可能占据的位置，而不只检查离散采样点。

## 4. 保守覆盖与安全裕度

### 4.1 误差预算

设地图边界误差、轨迹离散误差和闭环跟踪误差上界分别为

\[
\varepsilon_{\mathrm{map}},
\quad
\varepsilon_{\mathrm{disc}},
\quad
\varepsilon_{\mathrm{track}}.
\]

总位置误差取

\[
\varepsilon_p=
\varepsilon_{\mathrm{map}}+
\varepsilon_{\mathrm{disc}}+
\varepsilon_{\mathrm{track}}.
\]

模式生成时使用收缩后的有效传感器半宽

\[
r_k^{\mathrm{eff}}=
\frac{w_k}{2}-\varepsilon_p.
\]

机器人之间的规划距离要求为

\[
d_{\mathrm{plan}}=
d_{\mathrm{safe}}+2\varepsilon_p.
\]

### 定理 1：覆盖与安全裕度定理

若满足：

1. 机器人 \(k\) 的实际位置与标称轨迹任意时刻距离不超过 \(\varepsilon_p\)；
2. 标称轨迹以半径 \(r_k^{\mathrm{eff}}\) 的足迹覆盖其责任单元；
3. 两机器人标称轨迹任意同时刻的距离不少于 \(d_{\mathrm{safe}}+2\varepsilon_p\)；

则实际传感器足迹完整覆盖责任单元，且实际机器人间距离不少于 \(d_{\mathrm{safe}}\)。

证明：

对任意待覆盖点 \(x\)，存在标称轨迹点 \(\bar p\) 满足

\[
\|x-\bar p\|\leq
\frac{w_k}{2}-\varepsilon_p.
\]

由三角不等式和 \(\|p-\bar p\|\leq\varepsilon_p\)，有

\[
\|x-p\|
\leq
\|x-\bar p\|+\|\bar p-p\|
\leq\frac{w_k}{2},
\]

故实际足迹覆盖 \(x\)。对两机器人 \(k,l\)，

\[
\|p_k-p_l\|
\geq
\|\bar p_k-\bar p_l\|
-\|p_k-\bar p_k\|
-\|p_l-\bar p_l\|
\geq d_{\mathrm{safe}}.
\]

证毕。

该定理使覆盖与安全不再只依赖仿真采样点，而具有明确的误差裕度条件。

## 5. 时间扩展多模式图

### 5.1 图结构

对机器人 \(k\) 建立时间扩展模式图

\[
\mathcal G_k^T=
(\mathcal V_k^T,\mathcal E_k^T).
\]

节点包括：

- 起点节点 \(o_k\)；
- 终点节点 \(d_k\)；
- 节点 \((i,m,t)\)：机器人在时间槽 \(t\) 开始执行单元 \(i\) 的模式 \(m\)；
- 必要的等待节点。

边表示一个完整覆盖模式、连接原语或等待动作。时间槽长度为 \(\Delta t\)，所有边持续时间向上取整为整数槽数，从而保持保守可行性。

### 5.2 时空冲突资源

将自由空间分为安全资源区域 \(\mathcal Z\)，例如：

- 状态格顶点及其安全邻域；
- 有向运动边的扫掠管；
- 狭窄通道；
- 两条非完整运动原语的连续交叉管。

时空资源为

\[
h=(z,\tau)\in\mathcal H,
\]

其中 \(z\in\mathcal Z\)，\(\tau\) 为时间槽或连续时间段。容量 \(\operatorname{cap}_h\) 通常为 1；宽阔水域可大于 1。

若某条模式或连接的保守时空管占用资源 \(h\)，其占用系数为 1。对向通过狭窄航道、同向追越和连续交叉分别建立相应资源。

### 5.3 为什么采用资源容量割

路线对 no-good 割 \(x_r+x_{r'}\leq1\) 只排除一个路线组合，且可能错误禁止可通过重定时解决的几何路径。资源容量割

\[
\sum_{r} b_{hr}x_r\leq\operatorname{cap}_h
\]

描述的是产生冲突的真正原因。后续定价可以：

- 改变出发时间；
- 插入等待；
- 改换通道；
- 调整覆盖模式；
- 将单元转移给另一机器人。

因此同一个割能够引导尚未生成的新路线避开冲突。

## 6. 完整路线列

机器人 \(k\) 的一条路线列 \(r\in\mathcal R_k\) 是 \(\mathcal G_k^T\) 上从 \(o_k\) 到 \(d_k\) 的元素路径，且满足：

1. 每个责任单元至多访问一次；
2. 对访问的单元选择且只选择一个覆盖模式；
3. 时间沿路径单调增加；
4. 总能耗不超过 \(B_k\)；
5. 所有模式和连接都满足单机器人运动学与静态障碍约束。

路线属性为：

\[
a_{ir}=
\begin{cases}
1,&r\text{ 覆盖单元 }i,\\
0,&\text{否则},
\end{cases}
\]

\[
\tau_r=\text{路线完工时间},
\qquad
e_r=\text{路线总能耗},
\]

\[
b_{hr}=
\begin{cases}
1,&r\text{ 占用资源 }h,\\
0,&\text{否则}.
\end{cases}
\]

每个机器人还具有一条不覆盖任何单元的安全空路线，使机器人数量超过需要时主问题仍可行。

## 7. 词典序路线列主问题

### 7.1 第一阶段：最小化 makespan

二元变量 \(x_r\) 表示是否选择路线 \(r\)，连续变量 \(T\) 为舰队 makespan：

\[
\min T
\]

满足

\[
\sum_{r\in\mathcal R_k}x_r=1,
\qquad \forall k\in\mathcal K,
\tag{1}
\]

\[
\sum_{k\in\mathcal K}
\sum_{r\in\mathcal R_k}
a_{ir}x_r=1,
\qquad \forall i\in\mathcal I,
\tag{2}
\]

\[
\sum_{r\in\mathcal R_k}
\tau_r x_r\leq T,
\qquad \forall k\in\mathcal K,
\tag{3}
\]

\[
\sum_{k\in\mathcal K}
\sum_{r\in\mathcal R_k}
b_{hr}x_r\leq\operatorname{cap}_h,
\qquad \forall h\in\mathcal H',
\tag{4}
\]

\[
x_r\in\{0,1\}.
\tag{5}
\]

\(\mathcal H'\subseteq\mathcal H\) 是当前已经激活的冲突资源集合。能量预算已经内嵌于可行路线定义；也可以作为路线级约束显式保留。

约束 (2) 是责任精确一次，不禁止不同责任单元在边界处必要的传感器足迹重叠。

### 7.2 第二阶段：固定最优 makespan 后最小化能耗

设第一阶段最优值为 \(T^*\)。第二阶段求

\[
\min
\sum_k\sum_{r\in\mathcal R_k}e_r x_r
\]

满足约束 (1)—(5) 及

\[
\sum_{r\in\mathcal R_k}
\tau_r x_r\leq T^*,
\qquad \forall k.
\tag{6}
\]

最终目标是严格词典序对

\[
\left(
T_{\max},
E_{\mathrm{total}}
\right).
\]

转弯、逆流和等待通过物理时间与能耗进入目标，不再另设任意权重。

## 8. 列生成与定价

完整路线集合指数大，主问题只维护受限路线池

\[
\mathcal R_k'\subset\mathcal R_k.
\]

### 8.1 初始列

初始路线来自：

1. 当前论文的“分区 + 单艇 TSP + 避碰”路线；
2. MCFS/MMRTC 或 LS-MCPP 可行路线；
3. 按最小服务时间做的能力均衡贪婪路线；
4. 空路线。

把原算法路线放入初始池可确保新算法具有明确的保底可行解。

### 8.2 定价子问题

求受限主问题 LP 松弛，获得：

- 单机器人选路约束的对偶价 \(\alpha_k\)；
- 单元覆盖约束的对偶价 \(\pi_i\)；
- makespan 约束的对偶价 \(\eta_k\)；
- 冲突资源容量约束的对偶价 \(\lambda_h\)。

对机器人 \(k\)，定价子问题在 \(\mathcal G_k^T\) 上寻找约化成本为负的元素资源约束最短路。忽略符号约定后，路线的有效搜索代价由

\[
\bar c_r=
c_r^{\mathrm{stage}}
-\alpha_k
-\sum_i\pi_i a_{ir}
+\eta_k\tau_r
+\sum_{h\in\mathcal H'}\lambda_h b_{hr}
\tag{7}
\]

构成，其中第一阶段 \(c_r^{\mathrm{stage}}=0\)，第二阶段为 \(e_r\)。实际实现中对偶符号以主问题的标准形式为准。

式 (7) 具有明确含义：

- 高 \(\pi_i\) 鼓励新路线覆盖当前难分配单元；
- 高 \(\eta_k\) 抑制瓶颈机器人的长路线；
- 高 \(\lambda_h\) 抑制拥挤时空资源；
- 第二阶段直接权衡新增能耗。

### 8.3 标签状态

精确定价标签为

\[
L=
(v,t,e,S,\bar c),
\]

其中 \(v\) 是末节点，\(t\) 是时间槽，\(e\) 是能耗，\(S\) 是已经服务的单元集合，\(\bar c\) 是累计约化成本。

扩展规则：

1. 模式边只能在其单元不属于 \(S\) 时扩展；
2. 扩展后把该单元加入 \(S\)；
3. 时间与能量资源按边属性累加；
4. 冲突对偶价格按边占用的已激活资源累加；
5. 到达终点且约化成本为负时生成新列。

精确定价保留完整集合 \(S\)。大规模下可使用 ng-route 记忆放松，但只有证明该放松提供有效界时，才能把其结果称为下界。

### 8.4 安全支配

两个标签 \(L_1,L_2\) 只有在以下条件同时成立时才能支配：

- 到达相同的时间扩展节点；
- 具有相同的已访问集合，或使用经过证明的 ng 支配条件；
- \(L_1\) 的能量不大于 \(L_2\)；
- \(L_1\) 的约化成本不大于 \(L_2\)；
- \(L_1\) 不违反任何 \(L_2\) 可满足的分支决定。

本文不使用“访问集合更小就必然支配”之类未经证明的激进规则，因为单元对偶收益会使这类规则失效。

## 9. 惰性冲突资源分离

初始不枚举全部 \(\mathcal H\)，而只激活狭窄通道和明显交叉处的少量资源。每次受限主问题得到整数路线组合后：

1. 展开所有被选路线的连续时空管；
2. 用时空包围体筛选可能冲突的机器人对；
3. 对候选对计算保守时空管是否相交；
4. 若相交，定位其有限状态—时间冲突资源 \(h\)；
5. 将容量割 (4) 加入主问题；
6. 为未来列注册占用判定器，使定价列自动计算 \(b_{hr}\)；
7. 重新执行列生成和整数求解。

重要的是：新增冲突割后必须重新定价。若只重新求当前路线池的整数问题，冲突代价不能真正影响尚未生成的路线，就仍属于后处理。

### 9.1 连续冲突到有限资源的映射

所有模式和连接由有限运动原语构成。每个原语具有覆盖其整个持续时间的保守扫掠管。任意两条原语的管相交时，预先存在一个规范化冲突资源

\[
h=(g_1,g_2,\tau_1,\tau_2).
\]

资源割可写成两组占用列不能同时被选择，或将其投影到共同狭窄区域容量。因为运动原语、时间槽和机器人数量有限，潜在冲突资源集合 \(\mathcal H\) 有限。

## 10. 分支—定价—割

### 10.1 分支规则

优先采用与定价兼容的分支：

1. 单元—机器人分支：单元 \(i\) 必须或禁止由机器人 \(k\) 服务；
2. 单元相邻分支：在同一路线中强制或禁止 \(i\) 后直接访问 \(j\)；
3. 冲突先后分支：机器人 \(k\) 必须先于 \(l\) 占用特定狭窄资源。

所有分支决定都可转化为模式图节点、边或时间窗限制。

### 10.2 CROWN-BPC 完整流程

输入：单元集合、异构机器人、候选模式、时间分辨率和安全裕度。

输出：词典序最优路线组合，或带有效 Gap 的最好可行组合。

1. 生成经过覆盖和单机器人可行性验证的模式。
2. 建立每个机器人的时间扩展模式图。
3. 生成初始路线池与初始冲突资源。
4. 在当前分支节点求受限主问题 LP。
5. 对每个机器人精确定价，加入全部发现的负约化成本列。
6. 重复第 4—5 步，直到不存在负约化成本列。
7. 若 LP 解分数，选择兼容分支并递归。
8. 若获得整数解，执行连续时空管冲突分离。
9. 若发现新资源割，加入主问题并返回第 4 步。
10. 若无冲突，更新可行上界。
11. 第一阶段证明 \(T^*\) 后，加入约束 (6) 并以总能耗重新执行列生成、分支和冲突分离。
12. 输出两阶段上下界、Gap、路线与冲突验证证书。

## 11. 可扩展算法 CROWN-LNS

分支—定价—割最坏情况下仍是指数复杂。大规模实例使用与主问题共享路线、对偶和冲突资源的 CROWN-LNS。

### 11.1 只保留四类核心邻域

1. 瓶颈重定位：从当前 makespan 机器人移出一个单元链，尝试插入其他机器人；
2. 跨机器人链交换：交换两条路线的连续任务链，同时重算边界连接；
3. 模式—端点优化：固定责任和顺序，对相邻单元模式用动态规划联合选择；
4. 冲突资源重规划：选择高对偶价冲突资源附近的路线段，执行重定时、换通道或任务转移。

不再使用与核心无关的大量随机算子。

### 11.2 词典序真实边际

候选移动按

\[
\Delta=
\left(
\Delta T_{\max},
\Delta E_{\mathrm{total}}
\right)
\]

比较。只有第一项相同或不增加时才比较第二项。每次移动必须重新计算：

- 新模式服务时间；
- 前后连接轨迹；
- 能耗；
- 时空资源占用；
- 受影响机器人完工时间。

不允许用单元中心欧氏距离代替真实边际。

### 11.3 对偶引导

根节点或周期性路线池 LP 提供 \(\pi_i,\eta_k,\lambda_h\)：

- 高 \(\pi_i\)：难分配单元，优先重新组合；
- 高 \(\eta_k\)：当前瓶颈机器人；
- 高 \(\lambda_h\)：主要拥堵资源。

邻域抽样概率由这些对偶价格而非黑箱强化学习调整。

### 11.4 路线池整数重组

每当 LNS 产生一条新颖或低约化成本的机器人路线，就加入路线池。周期性求解池内主问题，使来自不同历史解的路线能够重新组合。

若根节点列生成已经精确收敛，其 LP 值是严格下界，LNS 任意时刻的最好无冲突解是上界。若只使用启发式定价，则必须标注为“经验下界”或不报告认证 Gap。

## 12. 理论性质

### 定理 2：冲突无感顺序规划不存在与团队规模无关的近似保证

考虑一类先忽略机器人间冲突选择各自最短路线、再固定几何路线做消冲突的顺序式算法。存在含 \(K\) 个机器人和 \(K\) 个责任单元的实例，使该算法消冲突后的 makespan 与联合最优 makespan 之比至少为

\[
\frac{K}{1+\epsilon},
\qquad \epsilon>0.
\]

因此该顺序式架构不存在与机器人数量 \(K\) 无关的常数近似比。

证明：

为每个机器人构造两条均能完成其责任单元的可行路线：

1. 公共路线 \(r_k^c\)，单机器人持续时间为 1，但在整个执行区间占用同一个容量为 1 的狭窄资源 \(h\)；
2. 私有路线 \(r_k^p\)，持续时间为 \(1+\epsilon\)，且不同机器人的私有路线互不冲突。

忽略机器人冲突时，因 \(1<1+\epsilon\)，每个机器人都严格选择公共路线。后处理必须将 \(K\) 条公共路线在资源 \(h\) 上串行化，故 makespan 为 \(K\)。联合优化直接为所有机器人选择私有路线，可获得 makespan \(1+\epsilon\)。于是比值为

\[
\frac{K}{1+\epsilon}.
\]

当 \(K\rightarrow\infty\) 时该比值无界，故不存在与团队规模无关的常数近似保证。证毕。

这个构造不说明每个顺序式实例都很差，而是说明仅凭单艇最短和最终避碰，无法获得舰队级质量保证。

### 定理 3：整数主问题与有限联合覆盖调度等价

假设：

1. \(\mathcal R_k\) 包含机器人 \(k\) 在有限时间扩展模式图上的全部可行元素路线；
2. \(\mathcal H\) 包含所有保守运动原语冲突资源；
3. 模式已经通过定理 1 所要求的覆盖裕度验证；

则主问题 (1)—(5) 的任一整数可行解与一个完成全部责任单元、满足能量约束且连续无碰撞的有限模型舰队调度一一对应。

证明：

正向：约束 (1) 为每个机器人选择一条完整起终路线；约束 (2) 使每个责任单元恰由一条被选路线服务；路线定义保证单机器人运动学和能量可行；约束 (4) 使任意冲突资源不超过容量。由保守管的定义，不共享超容量冲突资源意味着连续轨迹无碰撞；由定理 1，模式足迹覆盖对应单元。因此得到可行舰队调度。

反向：给定有限模式图上的可行舰队调度，每个机器人执行的节点—边序列对应 \(\mathcal R_k\) 中一条路线。选择这些路线后，机器人选路、责任覆盖和 makespan 约束显然成立；调度无碰撞意味着任何冲突资源容量均未违反。因此构成主问题整数可行解。证毕。

### 定理 4：惰性冲突分离的有限完备性

在定理 3 的有限模型下，若每次发现冲突都加入至少一个尚未激活且能排除该冲突的规范资源割，则惰性冲突分离经过至多 \(|\mathcal H|\) 次新增资源后终止；终止时所有整数候选均满足连续保守管安全约束。

证明：

\(\mathcal H\) 有限。每次冲突分离都加入一个此前不在 \(\mathcal H'\) 中的资源，因此 \(|\mathcal H'|\) 严格增加。最多加入 \(|\mathcal H|\) 个资源。若终止时仍存在碰撞，根据规范映射，该碰撞对应某个 \(h\in\mathcal H\)，分离器应发现并加入它，与终止条件矛盾。证毕。

该定理比路线对 no-good 割更强，因为它证明的是有限资源层面的完备性，而不是只排除已经见过的路线组合。

### 定理 5：CROWN-BPC 的词典序全局最优性

若：

1. 每个分支节点的定价子问题被精确求解；
2. 分支规则完备且与定价约束兼容；
3. 冲突分离满足定理 4；
4. 第一阶段和第二阶段均完整搜索；

则 CROWN-BPC 返回有限模式—时间模型上

\[
\left(
T_{\max},
E_{\mathrm{total}}
\right)
\]

的词典序全局最优解。

证明：

精确定价使每个节点的受限主问题 LP 等价于完整列主问题 LP；完备分支搜索保证获得满足所有整数约束的全局最优第一阶段解；定理 4 保证遗漏的冲突资源最终全部被分离，因此该解也是完整安全模型的最优 makespan \(T^*\)。第二阶段加入 \(T_{\max}\leq T^*\)，故其可行域恰为所有一级最优解；在该可行域内再次完整求解总能耗，得到词典序最优解。证毕。

### 定理 6：候选扫掠角离散误差界

设连续扫掠角可行域中的某个最优解为 \(S^*\)。固定 \(S^*\) 的机器人分配和单元访问顺序，机器人 \(k\) 的完整路线时间写成扫掠角向量的函数 \(T_k(\boldsymbol\beta)\)。假设在不改变访问拓扑和连接可行性的邻域内，逐坐标满足

\[
\left|
T_k(\ldots,\beta_i^1,\ldots)
-T_k(\ldots,\beta_i^2,\ldots)
\right|
\leq L_i|\beta_i^1-\beta_i^2|.
\]

若候选角集合对任意可行角的最大最近邻距离不超过 \(\Delta_\beta/2\)，则存在一个候选角解 \(\hat S\)，使

\[
T_{\max}(\hat S)
\leq
T_{\max}(S^*)
+
\max_{k}
\sum_{i\in R_k^*}
\frac{L_i\Delta_\beta}{2}.
\]

证明：

保持 \(S^*\) 的责任分配和访问顺序不变，将每个连续角 \(\beta_i^*\) 替换为最近候选角 \(\hat\beta_i\)。有

\[
|\hat\beta_i-\beta_i^*|
\leq\frac{\Delta_\beta}{2}.
\]

按机器人路线中的单元顺序逐一替换角度。由逐坐标 Lipschitz 条件和望远镜式累加，机器人 \(k\) 路线增加的时间不超过

\[
\sum_{i\in R_k^*}
L_i\frac{\Delta_\beta}{2}.
\]

对所有机器人取最大值即得结论。证毕。

该界只在访问拓扑不因角度微扰失效时成立。若角度改变导致通道连接性变化，应将对应角度分成不同可行区间，分别建立候选网格。

### 定理 7：服务工作量下界

令 \(s_{ikm}\) 为机器人 \(k\) 以模式 \(m\) 覆盖单元 \(i\) 的纯服务时间，不含连接、等待和返航。考虑连续分配松弛：

\[
\min L
\]

满足

\[
\sum_k\sum_{m\in\mathcal M_{ik}}
y_{ikm}=1,
\qquad \forall i,
\]

\[
\sum_i\sum_{m\in\mathcal M_{ik}}
s_{ikm}y_{ikm}\leq L,
\qquad \forall k,
\]

\[
0\leq y_{ikm}\leq1.
\]

其最优值 \(LB_{\mathrm{service}}\) 是真实最优 makespan 的有效下界。

证明：

任意可行整数舰队路线都为每个单元选择一个机器人和一个模式，因此对应上述松弛的一个整数可行点。该机器人真实路线时间至少等于其所有单元纯服务时间之和，因为连接、等待和返航均非负。连续松弛的最优值不大于任何整数可行点的最大服务时间，故不大于真实最优 makespan。证毕。

这里使用互不重叠责任单元的模式服务时间，而不是把传感器同时覆盖的微网格时间相加，因此避免原方案工作量下界可能重复计时的问题。

### 定理 8：anytime 最优性证书

令 \(LB_{\mathrm{CG}}\) 为经过精确定价收敛的根节点 LP 下界，\(LB_{\mathrm{service}}\) 为定理 7 下界，\(UB\) 为任意经过完整冲突验证的可行解，则

\[
LB=
\max
\{LB_{\mathrm{CG}},LB_{\mathrm{service}}\}
\leq OPT\leq UB.
\]

相对差距

\[
\operatorname{Gap}
=
\frac{UB-LB}{UB}\times100\%
\]

是一级 makespan 目标的有效最优性证书。

证明：

两个 \(LB\) 分量均为下界，其最大值仍不超过 \(OPT\)；可行解目标值是上界。代入定义即得。证毕。

### 推论 1：相同候选模型下不劣于顺序式基线

如果一个“固定分区—单艇 TSP—消冲突”基线的最终路线被加入 \(\mathcal R_k\)，则 CROWN-MCPP 的词典序最优解不劣于该基线。

该推论只作为实验保底性质，不列为主要理论贡献。

## 13. 复杂度与可扩展性声明

元素资源约束最短路和多机器人覆盖本身均为 NP-hard，因此：

- 完整路线列数量最坏为指数级；
- 精确定价最坏为指数级；
- 分支—定价—割不具有多项式时间保证；
- CROWN-LNS 不具有未经证明的固定近似比。

算法的可扩展性来自：

1. 只按需生成路线；
2. 冲突资源惰性分离；
3. 定价标签安全支配；
4. 根节点精确、深层节点启发式优先；
5. LNS 路线池重组；
6. 对模式和时间分辨率的分层精化。

论文必须同时报告时间、内存、列数、标签数、激活冲突资源数和 Gap，不能只报告最终路径长度。

## 14. 实验设计：证明“联合”而非证明“模块更多”

### 14.1 核心研究假设

- H1：在相同单元和模式库下，联合路线列优化显著优于固定分配后逐艇优化；
- H2：冲突资源反馈显著优于覆盖路径确定后的 MAPF 后处理；
- H3：多模式选择显著优于每个单元固定最小宽度方向；
- H4：CROWN-LNS 在大规模下接近 CROWN-BPC 最优解或有效下界；
- H5：保守覆盖/安全裕度能在存在跟踪误差时保持真实覆盖和最小间距。

### 14.2 基线

必须包括：

1. 当前论文原算法：分区 + 单艇 TSP + 现有冲突处理；
2. MIP-MMRTC；
3. LS-MCPP；
4. MCFS；
5. TMSTC*；
6. T-RO 2025 ESTC/LS-MCPP + path deconfliction；
7. CROWN-Post：相同模式与分配优化，但冲突只在最后处理；
8. CROWN-Fixed：固定任务分配，只优化单艇顺序和模式；
9. CROWN-NoMode：每个单元只保留一个方向；
10. CROWN-NoDual：LNS 不使用对偶价。

### 14.3 分层实例

小实例：

- 2—4 个机器人；
- 8—30 个责任单元；
- 完整 BPC、紧凑 MIP 或独立枚举交叉验证；
- 目标是证明最优性和实现正确性。

中实例：

- 3—12 个机器人；
- 30—200 个责任单元；
- 报告根节点 Gap、最终 Gap 和时间—质量曲线。

大实例：

- 10—100 个机器人；
- 200—2000 个责任单元；
- 固定墙钟预算下比较可行率、makespan 和相对有效下界的差距。

USV 实例：

- 岛群、狭窄水道、非凸岸线；
- 不同传感器幅宽、速度、转弯半径和电量；
- 无流、均匀流和已知时变流；
- 3—6 艘真实或半实物 USV 验证。

### 14.4 主要指标

- makespan；
- 总能耗；
- 一级最优性 Gap；
- 首个无冲突可行解时间；
- 冲突分离轮数；
- 后处理前后 makespan 增幅；
- 覆盖率和最小实际艇间距；
- 路线列数、定价标签数、激活资源数；
- 内存和墙钟时间。

### 14.5 关键图表

顶刊论文至少需要：

1. Sequential、Post-deconfliction 与 CROWN 的任务边界对比图；
2. 冲突割加入前后的路线归属变化图；
3. \(UB\)、\(LB\)、Gap 随时间曲线；
4. 机器人数量—求解时间/Gap 扩展曲线；
5. 单元模式与冲突资源的消融瀑布图；
6. 跟踪误差下覆盖率和最小间距验证图；
7. 真实 USV 时空轨迹与能量记录。

### 14.6 统计要求

- 随机或启发式算法每组至少 30 次独立运行；
- 相同实例、相同模式库、相同时间预算；
- 报告中位数、四分位距和 95% 置信区间；
- 使用配对 Wilcoxon 检验、Holm 校正和效应量；
- 精确算法超时必须保留其 \(LB\) 和 Gap；
- 无可行解、内存超限和冲突验证失败必须如实计数。

## 15. 借鉴来源与原创边界

| CROWN-MCPP 模块 | 借鉴来源与具体算法 | 本文新增部分 |
|---|---|---|
| 单元内部多模式 | [Revisiting Boustrophedon Coverage Path Planning as a GTSP](single/提取自978-981-15-9460-1.pdf) 的 E-GTSP；[复杂平面区域的无人机覆盖路径规划技术](merge/复杂平面区域的无人机覆盖路径规划技术.pdf) 的 FF/FR/RF/RR | 将模式嵌入异构机器人完整时间化路线列 |

| 海流模式成本 | [Energy efficient coverage path planning for USV-assisted inland bathymetry under current effects](single/1-s2.0-S0029801824012484-main.pdf) 的 \(E-\beta\) | 海流相关服务与连接成本进入路线列和词典序主问题 |
| 精确覆盖优化 | RA-L 2023 MIP-MMRTC | 从根树变量转为覆盖模式路线列，并加入时间资源与冲突定价 |

CROWN-MCPP 真正需要进行新颖性答辩的内容只有：

1. 时间扩展的异构覆盖模式路线列；
2. 可反馈到定价的保守冲突资源割；
3. CROWN-BPC 与 CROWN-LNS 共享模型和证书；
4. 针对该模型的定理 1—8。

其余组件必须明确标注为已有算法的适配或工程实现。

## 16. 投稿定位

### 16.1 ICAPS/AAAI 路线

突出：

- 新的组合优化模型；
- 定价与冲突分离；
- 定理 2—8；
- 公共二维 MCPP 基准；
- 与 MIP-MMRTC、LS-MCPP、MCFS 的严格比较。

USV 只作为一个高难应用实例，不能让论文退化为单一工程场景。

### 16.2 IEEE Transactions on Robotics 路线

除上述算法贡献外还必须增加：

- 连续非完整运动原语；
- 保守时空管安全验证；
- 至少 3—6 个实体机器人；
- 跟踪误差、定位误差和实际能量数据；
- 大规模仿真与公开实现。

### 16.3 海洋工程 SCI 一区路线

突出：

- 异构 USV；
- 海流相关能量；
- 岛群和狭窄航道；
- 湖试/海试；
- 相对于现有多 USV 顺序式方法的显著优势。

即便选择海洋工程期刊，也应保留下界和联合消冲突，因为这是论文区别于普通元启发式改进的关键。

## 17. 风险与审稿人可能提出的问题

### 风险 1：列生成只是标准技术

回答必须依靠新的定价图、覆盖模式资源、冲突资源对偶反馈和实验，而不是“第一次在 USV 上使用”。

### 风险 2：固定单元限制全局最优

承认结论相对于给定分解成立；通过细粒度单元、不同分辨率实验和定理 6 的模式角误差界分析近似损失。不要声称连续区域无条件全局最优。

### 风险 3：时间格太大

使用自适应时间分辨率：普通水域粗粒度，冲突热点局部细化。任何自适应细化都必须保持保守时间向上取整。

### 风险 4：冲突资源数量爆炸

用惰性分离、保守包围体筛选和资源聚合；同时报告激活资源比例，证明大多数潜在资源无需显式加入。

### 风险 5：大规模无法精确定价

明确区分：

- exact：精确定价并有严格 Gap；
- certified heuristic：根节点精确定价，整数部分由 LNS 求解；
- heuristic：仅启发式定价，不声称严格 Gap。

### 风险 6：多智能体性不足

本文的多智能体性来自共享时空资源和联合任务—路径决策，而不是分布式通信。若目标是 AAMAS，应另增分布式列生成或价格协商；若目标是 ICAPS、AAAI、ICRA、T-RO，集中式多机器人优化本身可以成立。

## 18. 论文最终应形成的逻辑链

\[
\text{固定责任单元}
\rightarrow
\text{异构单元多模式}
\rightarrow
\text{时间扩展完整路线列}
\rightarrow
\text{词典序集合划分主问题}
\rightarrow
\text{冲突资源定价与惰性分离}
\rightarrow
\text{精确或带证书的可扩展求解}.
\]

相较原 GLOBE-USV，CROWN-MCPP 少了在线故障、随机鲁棒、动态避碰和多分解搜索，但核心更深：

- 优化变量的语义更清楚；
- 冲突处理具有有限模型完备性；
- 下界不再重复计算覆盖工作量；
- 候选模式离散具有误差界；
- 精确解、启发式解和连续执行之间的声明边界明确；
- 每个主要创新都能对应一个独立消融和一个可检验定理。

因此，这一版本更接近多智能体顶刊顶会需要的“单一核心思想 + 完整数学闭环 + 明确理论性质 + 可扩展实证”，而不是一个覆盖很多场景但难以证明的综合系统。
