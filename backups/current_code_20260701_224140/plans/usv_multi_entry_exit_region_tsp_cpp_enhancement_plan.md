# 多入口/多出口区域 TSP-CPP 增强计划

## Summary
当前论文式区块扫描实验已经区分了“区块内往复扫描”和“区块间单旅行商问题”，但每个区块可供 TSP 选择的入口/出口模式仍然过少。结果是某些扫描线在候选图中存在，却因为唯一入口/出口连接不满足 3-DOF、转弯半径、边界或障碍约束而无法进入最终路线。

本计划把每个 `RegionCoveragePattern` 扩展为多入口、多出口候选：同一区块保持往复扫描覆盖结构不变，但允许改变扫描线访问方向、扫描线顺序和入口侧，使 TSP 在区块级访问顺序之外还能联合选择每个区块的入口/出口模式。

## Algorithm Changes
- 区块内部仍采用论文式往复扫描，扫描线端点只是覆盖路径点，不作为 TSP 节点。
- 每个区块生成多类候选：
  - 扫描轴候选：`x/y` 或区域首选轴与备用轴。
  - 扫描顺序候选：低坐标到高坐标、高坐标到低坐标。
  - 入口侧候选：第一条扫描线从最小端进入或从最大端进入。
  - pocket 候选：保留已有 `coverage_turn_pocket_scale` 多尺度安全口袋。
- 对每个候选重新计算：
  - 入口位姿、出口位姿。
  - 覆盖长度、内部 Dubins 转弯长度、总转角、估计时间。
  - 曲率、边界、障碍和动态可行性。
- TSP-CPP 求解时，每个区域节点不再只有一个模式，而是从动态可行候选集合中选择代价最低的入口/出口模式。

## Implementation Steps
1. 在 `PathPlanningConfig` 中新增多入口/多出口配置项。
2. 扩展论文式实验的 `_pattern_variants(...)`，由“原始 + 反向”升级为多扫描顺序、多入口侧组合。
3. 对每个变体重新编号 `CoveragePass`，并归一化航向。
4. 对候选模式重新计算 Dubins 内部转弯代价和 estimated time。
5. `_build_region_sweep_paths(...)` 按覆盖率、转弯边界裕度和时间排序，保留多个可行候选，而不是只保留一个代表模式。
6. `_solve_agent_region_tsp(...)` 记录每个区域候选数量、被拒绝连接和最终选中 pattern。
7. 可视化新增“已选中扫描模式”图，避免把全部候选模式误解为最终路线。
8. 报告中写入 `selected_patterns`、`candidate_pattern_counts` 和 `rejected_candidate_count`。

## Acceptance
- 对 15x15 小障碍地图，单个可行区块应出现多个入口/出口候选。
- 最终 07/08 图只展示 TSP 实际选中的扫描模式；04 图用于展示候选/代表扫描模式。
- 选中的区域间连接仍必须满足边界、障碍、曲率、航向连续和 3-DOF 动态验证。
- 若仍有区块无法连接，报告必须说明候选都失败的具体原因，而不是静默跳过。

## Notes
- 该增强对应论文中启发式/蚁群算法对区域入口、出口和区域访问顺序的联合优化思想。
- 本阶段不实现 ACO/FA3ACO，只补齐确定性多候选 TSP-CPP 框架，为后续替换元启发式算法保留接口。
