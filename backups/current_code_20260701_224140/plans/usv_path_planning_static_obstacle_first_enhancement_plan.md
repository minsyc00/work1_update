# 静态障碍优先的路径规划增强计划

## Summary

下一阶段先专注“矩形任务区域 + 多形状静态障碍物”的覆盖路径规划。动态障碍物暂不进入全局路径规划主流程，只保留接口和扩展点，后续再接入时空 A*/MAPF 与 CBF/RVO2/NMPC 联动。

## Key Changes

- 新增静态障碍模型，支持 `polygon`、`rectangle`、`circle`、`ellipse`，统一转换为多边形并按安全距离保守膨胀。
- 覆盖目标从完整矩形区域改为 `FreeSpace = MissionRectangle - InflatedStaticObstacles`。
- 区域分解升级为静态障碍感知分解，生成不进入障碍物的 `FreeSpaceCell`。
- 覆盖条带生成时裁剪障碍区间，条带中心线、矩形覆盖足迹、Dubins/Bezier 连接都必须避开膨胀障碍。
- A* 升级为静态障碍绕行搜索，代价包含路径长度、障碍邻域安全权重、航向变化、窄通道风险和边界风险。
- TSP-CPP 在自由空间子区域上运行，联合选择区域顺序、覆盖方向、入口/出口和无碰撞连接。
- MAPF/CBS 重新接入新路径规划层，重点处理窄通道、区域入口、转弯口袋和覆盖条带资源冲突。
- 残差补扫只针对自由空间漏扫，不把障碍内部计入覆盖残差。

## Implementation Plan

1. 扩展数据类型：新增 `StaticObstacle`、`ObstacleField`、`FreeSpaceCell`、`ObstacleAwareCoveragePattern`，并在路径规划请求中加入 `static_obstacles`。
2. 实现障碍归一化：矩形、多边形直接处理，圆/椭圆离散为多边形，所有障碍按 `d_safe` 和矩形足迹安全裕度膨胀。
3. 实现静态障碍碰撞检测：支持点、线段、采样 Dubins、采样 Bezier、矩形覆盖足迹与障碍的碰撞检查。
4. 实现自由空间分解：用任务矩形扣除膨胀障碍，基于障碍顶点做扫线/梯形分解，生成可覆盖 cell 和邻接图。
5. 实现障碍裁剪覆盖条带：覆盖线与障碍相交时切断，只保留自由空间内的合法 `CoveragePass`。
6. 升级区域图：节点为 `FreeSpaceCell`，边为可通行连接，边权使用无碰撞 A*/Dubins 连接代价。
7. 升级负载均衡：保持每艘艇分配区域连通，对绕障距离、窄通道和跨障碍连接增加代价。
8. 升级 A*：在自由空间通道图或栅格图上搜索绕障路径，安全权重随障碍邻域数量增加。
9. 升级 TSP-CPP：候选模式若条带或连接碰撞则过滤，区域间连接优先选择无碰撞 Dubins，不可行时使用 A* 通道连接。
10. 重新接入 MAPF/CBS：从新路径段生成 `PathRequirement`，对窄通道、入口、转弯区和覆盖条带做时窗冲突消解。
11. 完成静态障碍残差补扫闭环：检测自由空间漏扫，生成残差区域，重新分配、规划、调度并插入后续轨迹。
12. 保留动态障碍扩展接口：暂不改变主流程，只在类型和适配层保留未来 `DynamicObstacleTrack` 转时空约束的入口。

## Test Plan

- 障碍模型测试：多边形、矩形、圆形、椭圆障碍能正确归一化、离散和膨胀。
- 自由空间分解测试：生成的 cell 不与障碍相交，邻接关系正确。
- 覆盖测试：自由空间覆盖率达到 100%，障碍内部不计入覆盖目标。
- 碰撞测试：覆盖条带、Dubins 连接、Bezier 平滑段和矩形足迹均不穿越膨胀障碍。
- A* 绕障测试：障碍阻挡直线路径时，A* 能找到安全绕行路径。
- TSP-CPP 测试：区域访问顺序、覆盖模式、入口/出口均满足无碰撞约束。
- MAPF 测试：窄通道、区域入口和转弯口袋无资源时窗冲突。
- 残差补扫测试：人为制造障碍附近漏扫块后，能生成补扫任务并恢复自由空间完整覆盖。
- 集成测试：复杂静态障碍场景下 `PathPlanningLayer.plan_from_config(...)` 能输出可转换为 `TrajectoryReference` 的路径。

## Assumptions

- 当前阶段只实现静态障碍物全局路径规划。
- 动态障碍物作为后续阶段处理，不进入本阶段验收标准。
- 任务外边界仍为矩形，内部包含多个静态障碍。
- 障碍默认不可航行，覆盖目标只包含自由空间。
- 第一版优先使用项目内部几何实现，不强制新增外部几何依赖。
