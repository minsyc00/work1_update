# 多无人艇路径规划层总实现计划

## 目标

实现一套面向多无人艇矩形覆盖任务的论文融合路径规划层，逐步替换当前 bootstrap 逻辑，使默认 `paper_fusion_planner` 输出真实路径规划结果。

## 总体算法链

1. 建立路径规划层数据结构，表达区域、覆盖条带、候选覆盖模式、区域图、负载分配、单艇 TSP-CPP 路径和诊断指标。
2. 建立矩形覆盖足迹模型，计算条带间距 `Delta = wf * (1 - rho)` 和端部转弯缓冲 `H >= Rmin + lf / 2 + d_safe`。
3. 对任务区域进行分解，矩形区域优先按长边方向生成连续子区域，凹区域预留基于凹点和支撑线的分解接口。
4. 为每个子区域生成多个覆盖候选模式，每个模式包含条带序列、入口/出口位姿、覆盖长度、转弯长度、转弯角和曲率可行性。
5. 使用 Dubins 曲线计算 3-DOF 可跟踪的连接代价，并约束 `kappa <= 1 / Rmin + 1e-3`。
6. 构建区域图，节点为子区域，边为邻接或可通行连接，节点权重为最优覆盖时间，边权重为 Dubins 连接时间和航向变化代价。
7. 使用连通负载均衡图划分，把子区域分配给多艘艇，优化最大负载、负载差和连接代价。
8. 使用 turn-aware A* 为区域间连接和 TSP 初始顺序提供安全、短路径、少转弯的搜索代价。
9. 对每艘艇求解单艇 TSP-CPP，联合优化区域访问顺序、区域覆盖模式、入口和出口。
10. 使用 2-opt/3-opt 后优化减少总路径长度和总转弯角。
11. 将区域级路径组装成 `MultiAgentPathPlan`，并提供到 `SmoothedPath` 和 `TrajectoryReference` 的适配。
12. 接入 Dubins/Bezier 平滑和时间参数化，使输出轨迹可由 NMPC 跟踪。
13. 维护矩形足迹覆盖状态，检测残差区域，并提供残差补扫分配入口。
14. 增加单元测试和示例脚本，验证区域分解、覆盖、负载均衡、A*、TSP-CPP、曲率约束和轨迹适配。

## 验收标准

- `PathPlanningLayer.plan_from_config(...)` 默认运行真实 `paper_fusion_planner`。
- 所有覆盖条带满足矩形足迹无漏扫要求。
- 所有转弯和区域间连接满足最小转弯半径约束。
- 多艇负载分配保持连通，默认负载差目标不超过 10%，区域粒度不足时输出诊断。
- 单艇 TSP-CPP 后优化结果不差于初始解。
